# Administration

![Brugermenu med administrationsmuligheder](../assets/img/en/19_user_menu.png)

**Admin**-sektionen er tilgængelig for brugere med **Admin**-rollen. Den tilbyder værktøjer til at konfigurere platformens datamodel, brugeradgang, integrationer og systemdækkende indstillinger.

| Side | Beskrivelse |
|------|-------------|
| [Generelle indstillinger](settings.md) | Udseende (logo, favicon, valuta), aktiverede sprog, regnskabsår, e-mail (SMTP) og modulskifter (BPM, PPM, GRC) |
| [Metamodel](metamodel.md) | Korttyper, felter, sektioner, undertyper, relationstyper, beregninger, tags, EA-principper og metamodel-grafen |
| [Beregninger](calculations.md) | Administratordefinerede formler, der automatisk beregner feltværdier, når kort gemmes |
| [Tags](tags.md) | Taggrupper og tags til fleksibel kategorisering af kort |
| [Brugere og roller](users.md) | Brugeradministration, app-niveau-roller og tilladelseskonfiguration |
| [Godkendelse og SSO](sso.md) | Single Sign-On-konfiguration (Microsoft Entra ID, Google, Okta, Generic OIDC) og invitationsarbejdsprocesser |
| [AI-funktioner](ai.md) | AI-beskrivelsesforslag drevet af LLM'er (Ollama, OpenAI-kompatible udbydere) med websøgningskontekst |
| [MCP-integration](mcp.md) | Model Context Protocol-server til AI-værktøjsadgang (Claude Desktop, Copilot, Cursor) |
| [API-reference](api.md) | Live OpenAPI 3-spec indlejret i brugermanualen (ingen backend-instans påkrævet), plus godkendelse, tilladelser og kodegenererings-tips |
| [End-of-Life (EOL)](eol.md) | Massiv EOL-datakobling fra endoflife.date til teknologikort |
| [Undersøgelser](surveys.md) | Datavedligeholdelsesundersøgelser rettet mod specifikke korttyper og interessenter |
| [Webportaler](web-portals.md) | Offentlige, skrivebeskyttede portaler tilgængelige uden godkendelse |
| [ServiceNow-integration](servicenow.md) | Tovejssynkronisering mellem Turbo EA-kort og ServiceNow CMDB |
