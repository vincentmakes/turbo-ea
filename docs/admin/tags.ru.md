# Теги

Функция **Теги** (**Администрирование > Метамодель > вкладка «Теги»**) позволяет создавать классификационные метки, которые пользователи могут применять к карточкам. Теги организованы в **группы тегов**, каждая из которых имеет собственный режим выбора, ограничения по типам и необязательный флаг обязательности, участвующий в рабочем процессе утверждения и в оценке качества данных.

## Группы тегов

Группа тегов — это категория тегов. Например, можно создать группы «Бизнес-домен», «Стандарт соответствия» или «Ответственная команда».

### Создание группы тегов

Нажмите **+ Новая группа тегов** и настройте:

| Поле | Описание |
|------|----------|
| **Название** | Отображаемое имя в карточке, фильтрах инвентаря и отчётах. |
| **Описание** | Необязательный свободный текст, видимый только администраторам. |
| **Режим** | **Одиночный выбор** — один тег на карточку. **Множественный выбор** — несколько тегов на карточку. |
| **Обязательно** | Если флажок установлен, группа участвует в шлюзе утверждения и в оценке качества для каждого типа карточек, к которому она применяется. Подробнее см. раздел **Обязательные группы тегов** ниже. |
| **Ограничить по типам** | Необязательный список разрешённых типов карточек. Если пусто — группа доступна для всех типов; иначе её видят в деталях, фильтрах и порталах только перечисленные типы. |

### Управление тегами

Внутри каждой группы можно добавлять отдельные теги:

1. Нажмите **+ Добавить тег** внутри группы тегов.
2. Введите **название** тега.
3. При желании задайте **цвет** для визуального различия — цвет определяет фон чипа в карточке, инвентаре, отчётах и веб-порталах.

Теги отображаются на страницах карточек в секции **Теги**, где пользователи с соответствующими правами могут их применять или удалять.

## Ограничения по типам

Установка **Ограничить по типам** для группы тегов сужает её область применения сразу во всех местах:

- **Карточка** — группа и её теги показываются только на подходящих типах карточек.
- **Боковая панель фильтров инвентаря** — чип группы появляется в `TagPicker` только когда инвентарь отфильтрован по подходящему типу.
- **Веб-порталы** — группа показывается читателям портала только тогда, когда портал представляет подходящий тип.
- **Отчёты** — выпадающие списки группировки / фильтрации включают группу только для подходящих типов.

Админ-интерфейс отображает назначенные типы в виде небольших чипов на каждой группе, чтобы область действия была видна с первого взгляда.

## Обязательные группы тегов

Отметка группы как **Обязательной** превращает её в требование управления: каждая карточка, к которой группа применяется, должна содержать хотя бы один тег из этой группы.

### Шлюз утверждения

Карточка не может перейти в состояние **Утверждено**, пока не удовлетворены все применимые обязательные группы тегов. Попытка утверждения возвращает ошибку `approval_blocked_mandatory_missing`, а страница карточки перечисляет недостающие группы. Две настройки сохраняют шлюз безопасным:

- Группа применяется к карточке только если её список **Ограничить по типам** пуст или содержит тип карточки.
- Обязательная группа, в которой **ещё не настроено ни одного тега**, тихо пропускается — это предотвращает недостижимый шлюз утверждения из-за неполной настройки администратором.

Как только добавлены необходимые теги, карточку можно утверждать в обычном порядке.

### Вклад в качество данных

Применимые обязательные группы тегов также участвуют в оценке качества данных карточки. Каждая удовлетворённая группа повышает оценку наравне с другими обязательными элементами (обязательные поля, обязательные стороны связей), составляющими расчёт полноты.

### Визуальные индикаторы

Обязательные группы помечаются чипом **Обязательно** как в административном списке, так и в секции «Теги» в карточке. Недостающие обязательные теги показываются в баннере статуса утверждения и во всплывающей подсказке кольца качества данных, чтобы пользователь точно знал, что нужно добавить.

## Разрешения

| Разрешение | Что оно даёт |
|------------|--------------|
| `tags.manage` | Создавать, редактировать и удалять группы тегов и теги в административном интерфейсе, а также применять / удалять теги на любой карточке независимо от других разрешений. |
| `inventory.edit` + `card.edit` | Применять или удалять теги на карточках, которые пользователь вправе редактировать (через прикладную роль или роль заинтересованного лица на конкретной карточке). |

`tags.manage` по умолчанию предоставляется роли admin. `inventory.edit` принадлежит ролям admin, bpm_admin и member; `card.edit` назначается через распределения ролей заинтересованных лиц у самой карточки.

Зрители (viewers) **видят** теги, но не могут их менять.

## Где появляются теги

- **Карточка** — секция «Теги» перечисляет применимые группы и текущие теги. Обязательные группы показывают чип; ограниченные группы появляются только когда тип карточки совпадает.
- **Боковая панель фильтров инвентаря** — сгруппированный `TagPicker` позволяет фильтровать сетку инвентаря по одному или нескольким тегам. Группы и теги отфильтрованы по текущему типу.
- **Отчёты** — нарезка по тегам доступна в портфельных, матричных и других отчётах, поддерживающих измерения группировки / фильтрации.
- **Веб-порталы** — редакторы порталов могут предоставлять фильтры по тегам анонимным читателям, чтобы внешние потребители нарезали публичные ландшафты тем же способом.
- **Диалоги создания / редактирования** — тот же `TagPicker` появляется при создании новой карточки, позволяя задавать обязательные теги с самого начала — это особенно полезно для обязательных групп.
