# Tags

La fonctionnalité **Tags** (**Admin > Métamodèle > onglet Tags**) vous permet de créer des étiquettes de classification que les utilisateurs peuvent appliquer aux fiches. Les tags sont organises en **groupes de tags**, chacun avec son propre mode de sélection et ses restrictions de type.

## Groupes de tags

Un groupe de tags est une catégorie de tags. Par exemple, vous pourriez créer des groupes comme « Domaine Métier », « Cadre de Conformite » ou « Propriété par Équipe ».

### Création d'un groupe de tags

Cliquez sur **+ Nouveau groupe de tags** et configurez :

| Champ | Description |
|-------|-------------|
| **Nom** | Nom d'affichage du groupe |
| **Mode** | **Sélection unique** (un tag par fiche) ou **Sélection multiple** (plusieurs tags par fiche) |
| **Restreindre aux types** | Optionnellement limiter quels types de fiches peuvent utiliser ce groupe de tags. Si vide, le groupe est disponible pour tous les types |

### Gestion des tags

Au sein de chaque groupe, vous pouvez ajouter des tags individuels :

1. Cliquez sur **+ Ajouter un tag** dans un groupe de tags
2. Entrez le **nom** du tag
3. Optionnellement définissez une **couleur** pour la distinction visuelle

Les tags apparaissent sur les pages de détail des fiches dans la section **Tags**, ou les utilisateurs peuvent les appliquer ou les retirer.

## Utilisation des tags

- **Détail de la fiche** -- Les utilisateurs voient les groupes de tags disponibles et peuvent appliquer des tags depuis la section Tags
- **Filtrage de l'inventaire** -- La barre laterale de filtres de l'inventaire inclut le filtrage par tags
- **Rapports** -- Les tags peuvent être utilises comme critères de regroupement ou de filtrage dans les rapports
- **Portails web** -- La visibilité des tags peut être activée par portail
