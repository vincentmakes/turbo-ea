# Tags

O recurso de **Tags** (**Admin > Metamodelo > aba Tags**) permite criar rótulos de classificação que usuários podem aplicar a cards. Tags são organizadas em **grupos de tags**, cada um com seu próprio modo de seleção e restrições de tipo.

## Grupos de Tags

Um grupo de tags é uma categoria de tags. Por exemplo, você pode criar grupos como "Domínio de Negócio", "Framework de Conformidade" ou "Propriedade da Equipe".

### Criando um Grupo de Tags

Clique em **+ Novo Grupo de Tags** e configure:

| Campo | Descrição |
|-------|-----------|
| **Nome** | Nome de exibição para o grupo |
| **Modo** | **Seleção única** (uma tag por card) ou **Seleção múltipla** (múltiplas tags por card) |
| **Restringir a Tipos** | Opcionalmente limite quais tipos de card podem usar este grupo de tags. Se vazio, o grupo está disponível para todos os tipos |

### Gerenciando Tags

Dentro de cada grupo, você pode adicionar tags individuais:

1. Clique em **+ Adicionar Tag** dentro de um grupo de tags
2. Insira o **nome** da tag
3. Opcionalmente defina uma **cor** para distinção visual

Tags aparecem nas páginas de detalhe dos cards na seção **Tags**, onde usuários podem aplicá-las ou removê-las.

## Como as Tags São Usadas

- **Detalhe do card** — Usuários veem os grupos de tags disponíveis e podem aplicar tags na seção de Tags
- **Filtragem do inventário** — A barra lateral de filtros do inventário inclui filtragem baseada em tags
- **Relatórios** — Tags podem ser usadas como critério de agrupamento ou filtragem em relatórios
- **Portais web** — A visibilidade de tags pode ser habilitada por portal
