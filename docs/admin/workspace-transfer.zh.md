# 工作区迁移

工作区迁移（**管理 → 设置 → 迁移 → 工作区迁移**）将一个完整的 Turbo EA 工作区作为单个自包含的捆绑包，从一个实例迁移到另一个实例。其核心使用场景是：你在**本地**实例上构建好一个工作区，然后需要将所有内容推送到**生产环境**。

![工作区迁移](../assets/img/zh/58_workspace_transfer.png)

## 包含的内容

导出会将完整工作区打包为一个 `.zip` 捆绑包，其中包含一个 Excel 工作簿（所有结构化数据，每个域一张工作表），并在相关情况下包含一个用于存放非结构化文件的 `assets/` 文件夹：

- **元模型** — 卡片类型和关系类型，包括所有自定义字段、子类型、区段和翻译。
- **配置** — 角色、每类型的相关方角色、标签组和标签、计算字段、EA 原则以及合规法规。
- **设置** — 货币、日期格式、功能开关、登录品牌、已启用的语言区域，以及其余的通用应用设置。
- **用户** — 邮箱、显示名称、角色和活动标志（用于在目标实例上重新关联所有权和分配关系）。不包含密码或 SSO 身份。
- **清单** — 每张卡片（含其层级、生命周期和属性）、卡片标签以及关系。
- **卡片上下文** — 相关方、文档链接、评论、待办事项和文件附件。
- **模块数据** — BPM（流程图、元素、流程版本、评估）、PPM（状态报告、成本、预算、风险、任务、WBS、依赖关系）、GRC 风险登记册（风险、缓解任务及其发生记录、卡片链接）、GRC 合规发现（连同它们所引用的分析运行记录）、架构决策与架构工作说明书（Statements of Architecture Work）、自由绘制图表、已保存报表、书签（清单的已保存视图，包括其共享设置）、Web 门户以及调查问卷。
- **资产** — 二进制文件附件、图表和 BPMN XML，以及徽标/网站图标，均作为独立文件存放在捆绑包的 `assets/` 文件夹内。

## 永不包含的内容

出于安全考虑，**密钥永不导出**：

- SMTP 密码
- SSO 客户端密钥
- AI 提供商 API 密钥
- ServiceNow 凭据

导入后，你必须在目标实例上重新输入这些内容。这是设计使然且无法避免：加密值与源实例的 `SECRET_KEY` 绑定，无法在任何其他位置解密。

还有一些内容会因设计而留在原地：

- **TurboLens 分析结果**（供应商分析、重复项聚类、现代化评估、已保存的架构评估）以及仪表板的 KPI 历史记录是实例本地数据 — 请在目标实例上重新运行这些分析。合规发现是例外，会随迁移一并传输。
- **浏览器本地状态**永不迁移：清单表格的临时列排序保存在浏览器的本地存储中，而不在数据库里。你**在已保存视图中**保存的列布局则会随该视图一起迁移。

## 导出

1. 打开 **管理 → 设置 → 迁移 → 工作区迁移**。
2. （可选）勾选**包含已归档卡片**，将已归档清单加入捆绑包。
3. 点击**导出捆绑包**。浏览器会下载 `workspace_export_<timestamp>.zip`。

## 导入

1. 在**目标**实例上，打开 **管理 → 设置 → 迁移 → 工作区迁移**。
2. 在**导入工作区**下，点击**选择捆绑包…**并选中你导出的 `.zip` 文件。
3. Turbo EA 解析该捆绑包并显示**试运行预览** — 一张分区段的表格，列出将会创建、更新、跳过或处于冲突状态的实体数量。此时尚未写入任何内容。
4. 审阅预览，然后点击**应用导入**。

导入是**幂等的**：元模型和配置按 key 匹配，卡片按外部 id 或按类型 + 层级路径匹配，用户按邮箱匹配。重新导入相同的捆绑包是安全的 — 已存在的实体会被跳过而非重复创建。已存在的 built-in 元模型类型保留其身份；只有其可编辑的 schema 会被合并。

### 解读预览

- **「跳过」意味着「已存在 — 无需任何操作」。** 在全新安装上，你通常会看到 Turbo EA 自带内容（相关方角色、资源类型、默认设置）被跳过，因为捆绑包中的副本与目标实例已有的内容完全相同。展开某个区段行（左侧箭头）可查看按原因划分的明细，以及任何冲突或失败消息。
- **版本提示。** 预览会显示捆绑包是从哪个 Turbo EA 版本导出的，并在其与导入实例版本不同时发出警告。该警告仅供参考 — 导入仍会执行 — 但在同一版本上导出和导入是最稳妥的做法。

## 导入之后

- 在各自的设置标签页下重新输入所有 SMTP、SSO 和 AI 凭据。
- 捆绑包所引用的合成用户落地时为**停用**状态；按需在 **管理 → 用户** 中将其激活。
- **用户拥有的数据跟随用户，通过邮箱匹配。** 待办事项、已保存视图、收藏以及其他个人数据都归属于邮箱与捆绑包中一致的账户。如果你在目标实例上使用与源实例不同的邮箱登录，你的个人条目会看起来像丢失了 — 它们其实挂在（可能处于停用状态的）匹配账户上。请使用相同的邮箱登录，或在 **管理 → 用户** 中激活该匹配账户。
- 私有的已保存视图仅对其所有者可见；共享视图和公开视图遵循各自的可见性设置。

## 重新开始

系统没有内置的「撤销导入」功能。要重置目标实例并从头重新导入，请使用 `RESET_DB=true` 重启一次（删除并重建所有表，然后重新写入种子数据），随后务必在下一次重启**之前**将其改回 `RESET_DB=false`，以免抹掉刚刚导入的数据。

## 权限

工作区迁移由两个专用权限控制，两者均授予管理员：

- `admin.export_workspace` — 导出捆绑包。
- `admin.import_workspace` — 预览并应用导入。
