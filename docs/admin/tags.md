# Tags

The **Tags** feature (**Admin > Metamodel > Tags** tab) lets you create classification labels that users can apply to cards. Tags are organized into **tag groups**, each with its own selection mode and type restrictions.

## Tag Groups

A tag group is a category of tags. For example, you might create groups like "Business Domain", "Compliance Framework", or "Team Ownership".

### Creating a Tag Group

Click **+ New Tag Group** and configure:

| Field | Description |
|-------|-------------|
| **Name** | Display name for the group |
| **Mode** | **Single select** (one tag per card) or **Multi select** (multiple tags per card) |
| **Restrict to Types** | Optionally limit which card types can use this tag group. If empty, the group is available on all types |

### Managing Tags

Within each group, you can add individual tags:

1. Click **+ Add Tag** inside a tag group
2. Enter the tag **name**
3. Optionally set a **color** for visual distinction

Tags appear on card detail pages under the **Tags** section, where users can apply or remove them.

## How Tags Are Used

- **Card detail** — Users see available tag groups and can apply tags from the Tags section
- **Inventory filtering** — The inventory filter sidebar includes tag-based filtering
- **Reports** — Tags can be used as grouping or filtering criteria in reports
- **Web portals** — Tag visibility can be enabled per portal
