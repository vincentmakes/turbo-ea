# Опросы

Модуль **Опросы** (**Администрирование > Опросы**) позволяет администраторам создавать **опросы для поддержания данных**, которые собирают структурированную информацию от заинтересованных сторон о конкретных карточках.

## Сценарии использования

Опросы помогают поддерживать актуальность ваших архитектурных данных, обращаясь к людям, наиболее близким к каждому компоненту. Например:

- Ежегодно запрашивать у владельцев приложений подтверждение бизнес-критичности и дат жизненного цикла
- Собирать оценки технической пригодности от ИТ-команд
- Получать обновления по затратам от владельцев бюджетов

## Жизненный цикл опроса

Каждый опрос проходит через три состояния:

| Статус | Значение |
|--------|----------|
| **Черновик** | Разрабатывается, ещё не виден респондентам |
| **Активный** | Открыт для ответов, назначенные заинтересованные стороны видят его в своих задачах |
| **Закрыт** | Больше не принимает ответы |

## Создание опроса

1. Перейдите в **Администрирование > Опросы**
2. Нажмите **+ Новый опрос**
3. Откроется **Конструктор опросов** со следующими настройками:

### Целевой тип

Выберите, к какому типу карточек применяется опрос (например, Приложение, ИТ-компонент). Опрос будет отправлен для каждой карточки этого типа, соответствующей вашим фильтрам.

### Фильтры

При необходимости сузьте область действия, фильтруя карточки (например, только активные приложения, только карточки, принадлежащие определённой организации).

### Вопросы

Составьте ваши вопросы. Каждый вопрос может быть:

- **Свободный текст** — развёрнутый ответ
- **Одиночный выбор** — выбор одного варианта из списка
- **Множественный выбор** — выбор нескольких вариантов
- **Число** — числовой ввод
- **Дата** — выбор даты
- **Логический** — переключатель Да/Нет

### Автоматические действия

Настройте правила, которые автоматически обновляют атрибуты карточки на основе ответов опроса. Например, если респондент выбирает «Критически важный» для бизнес-критичности, поле `businessCriticality` карточки может быть обновлено автоматически.

## Отправка опроса

Когда ваш опрос в статусе **Активный**:

1. Нажмите **Отправить** для рассылки опроса
2. Для каждой целевой карточки создаётся задача для назначенных заинтересованных сторон
3. Заинтересованные стороны видят опрос на вкладке **Мои опросы** на [странице задач](../guide/tasks.ru.md)

## Просмотр результатов

Перейдите в **Администрирование > Опросы > [Название опроса] > Результаты**, чтобы увидеть:

- Статус ответов по каждой карточке (ответил, ожидает)
- Индивидуальные ответы с ответами на каждый вопрос
- Действие **Применить** для фиксации правил автоматических действий в атрибутах карточки
