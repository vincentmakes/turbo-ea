# 元模型

**元模型**定义了平台的整个数据结构 —— 存在哪些卡片类型、它们有哪些字段、它们如何相互关联以及卡片详情页面的布局方式。一切都是**数据驱动的**：您通过管理 UI 配置元模型，而不是更改代码。

![元模型配置](../assets/img/zh/20_admin_metamodel.png)

导航到**管理 > 元模型**访问元模型编辑器。它有七个标签页：**卡片类型**、**关系类型**、**计算**、**标签**、**元模型图**、**EA 原则** 和 **合规法规**。

## 卡片类型

卡片类型标签页列出系统中的所有类型。Turbo EA 内置 14 种类型，分布在四个架构层：

| 层 | 类型 |
|----|------|
| **战略与转型** | 目标、平台、项目 |
| **业务架构** | 组织、业务能力、业务上下文、业务流程 |
| **应用与数据** | 应用程序、接口、数据对象 |
| **技术架构** | IT 组件、技术类别、供应商、系统 |

### 创建自定义类型

点击 **+ 新建类型**创建自定义卡片类型。配置：

| 字段 | 描述 |
|------|------|
| **键** | 唯一标识符（小写，无空格）—— 创建后不可更改 |
| **标签** | 在 UI 中显示的名称 |
| **图标** | Google Material Symbol 图标名称 |
| **颜色** | 类型的品牌颜色（用于清单、报告和图表） |
| **类别** | 架构层分组 |
| **支持层级** | 此类型的卡片是否可以有父子关系 |

### 编辑类型

点击任何类型打开**类型详情面板**。在这里您可以配置：

#### 类型颜色

每个卡片类型（包括内置类型）都有一个可自定义的颜色，用于清单、报表、依赖视图和图表。这样可以让 Turbo EA 与组织的视觉规范保持一致（例如 TOGAF/ArchiMate 配色：业务元素用黄色/橙色，应用用蓝色）。

- 使用抽屉中的色块选择颜色。当所选颜色在浅色或深色背景下对比度过低时，会显示提示。
- 当内置类型的颜色与 Turbo EA 默认值不同时，色块旁会显示**重置**按钮，随时可以恢复标准配色。
- 显示在类型颜色上的文字（标签、图表形状）会自动在黑白之间切换以保证可读性，浅色和深色模式均适用。
- 选择器会在调色板旁边显示**实时预览**：类型名称、标签、卡片图标、子类型、卡片 ID 徽章以及依赖视图节点，分别以浅色和深色模式呈现，并随选择实时更新。

#### 字段

字段定义此类型卡片上可用的自定义属性。每个字段具有：

| 设置 | 描述 |
|------|------|
| **键** | 唯一的字段标识符 |
| **标签** | 显示名称 |
| **类型** | text、multiline_text、number、cost、boolean、date、url、single_select 或 multiple_select |
| **选项** | 对于选择字段：可用的选项及其标签和可选颜色 |
| **必填** | 字段是否必须填写（用于数据质量评分） |
| **数据质量** | 每个字段对评分的贡献在**数据质量**面板中管理（见下文） |
| **只读** | 禁止手动编辑（适用于计算字段） |

点击 **+ 添加字段**创建新字段，或点击现有字段在**字段编辑器对话框**中编辑。

#### 分区

字段在卡片详情页面上按**分区**组织。您可以：

- 创建命名分区来分组相关字段
- 将分区设置为 **1 列**或 **2 列**布局
- 在分区内将字段组织为**组**（渲染为可折叠的子标题）
- 在分区内通过拖拽重新排序字段，并通过字段的**移动**操作将其移至其他分区

特殊分区名称 `__description` 将字段添加到卡片详情页面的描述分区。

#### 卡片 ID

打开**卡片 ID 生成**开关，即可为此类型的卡片提供稳定、易读的 ID（例如 `APP-00001`）。该 ID 会以可点击复制的标签显示在卡片类型旁边，作为清单中的可选列（可排序、可筛选）、出现在 Excel 导出中，并可在计算字段公式中使用（通过 `data.reference`）。

**编号始终自动生成**，你只能控制**前缀**。开启时会以文本形式显示一个建议前缀（根据类型名称推导，例如 `APP-`）——点击铅笔即可修改。两项设置用于调整编号：

- **起始编号** — 序列的第一个编号（默认 `1`）。
- **最少位数** — 补零宽度（默认 `5`），因此 `1` 显示为 `00001`。这是最小值；一旦超过便会加长。**示例**会随输入实时显示第一个 ID。

ID 在全局范围内**唯一、只读，且永不重复使用或更改**。编号序列**按前缀在整个工作区内**跟踪，因此共享同一前缀的两个类型会形成一个连续、无冲突的序列。一旦此类型的卡片拥有 ID，整个格式（前缀、起始编号和最少位数）即被锁定（字段变为只读）；但仍可关闭生成。保存永远不会为现有卡片分配 ID；请使用专门的**生成 ID**按钮按需补齐（带进度条和确认）。

#### 数据质量评分

卡片的**数据质量**评分以加权方式衡量其完整程度。每个参与计分的因素——每个字段以及五个内置因素——都在同一处管理：卡片类型编辑器中的**数据质量**选项卡。（编辑器分为多个选项卡——主要、关系、干系人角色和数据质量——翻译可通过标题栏中的图标访问。）

每个因素的重要性通过一个简单的四档滑块设置，同时显示其对应的数字：

- **忽略 (0)**——完全从评分中排除。
- **普通 (1)**——计一次（默认）。
- **重要 (2)**——计两倍。
- **关键 (3)**——计三倍。

该面板先列出五个**内置因素**——**描述**、**生命周期**（是否设置了任意生命周期日期）、**必填关系**、**必填标签**和**干系人角色**（为该类型定义的每个角色在指派干系人后即视为完成）——随后是按分区分组的每个字段，均使用相同的滑块。例如，对于其卡片本就从不包含日期的类型，可将**生命周期**设为*忽略*，以免它们被扣分。

面板顶部的**评分构成**条显示每个因素在最高可能评分中所占的比例，让您一眼看出哪些因素占主导。在**主要**选项卡的卡片布局中，每个字段——以及内置的描述、生命周期和关系分区——都会显示一个标注其当前档位数字的小徽标，让您无需离开该选项卡即可查看权重。

更改任何重要性都会立即对该类型的所有现有卡片重新评分。新字段默认为*普通*，因此添加后即计入评分。

#### 子类型（子模板）

子类型充当卡片类型内的**子模板**。每个子类型可以控制该子类型的卡片显示哪些字段，而所有字段仍然在卡片类型级别定义。

例如，应用程序类型有子类型：业务应用、微服务、AI 代理和部署。管理员可以为 SaaS 子类型隐藏与服务器相关的字段，因为这些字段不相关。

**按子类型配置字段可见性：**

1. 在元模型管理中打开一个卡片类型。
2. 点击任意子类型标签以打开**子类型模板**对话框。
3. 使用开关切换字段可见性 —— 关闭的字段将对该子类型的卡片隐藏。
4. 隐藏的字段不计入数据质量评分，因此用户不会因看不到的字段而被扣分。

当卡片未选择子类型（或该类型没有子类型）时，所有字段均可见。隐藏的字段会保留其数据 —— 如果卡片的子类型发生变化，之前隐藏的值会被保留。

#### 干系人角色

为此类型定义自定义角色（例如「应用所有者」、「技术所有者」）。每个角色携带**卡片级权限**，在访问卡片时与用户的应用级角色组合。有关权限模型的更多信息，请参阅[用户与角色](users.md)。

#### 翻译

点击类型抽屉工具栏中的**翻译**按钮，打开**翻译对话框**。在这里，您可以为每种支持的语言提供所有元模型标签的翻译：

- **类型标签** — 卡片类型的显示名称
- **子类型** — 每个子类型的标签
- **分区** — 卡片详情页面上的分区标题
- **字段** — 字段标签和选择选项标签
- **利益相关者角色** — 在利益相关者分配界面中显示的角色名称

翻译与每个卡片类型一起存储，并在渲染时根据用户选择的语言进行解析。未翻译的标签将回退到英语默认值。

### 删除类型

- **内置类型**被软删除（隐藏），可以恢复
- **自定义类型**被永久删除

## 关系类型

关系类型定义卡片类型之间允许的连接。每个关系类型指定：

| 字段 | 描述 |
|------|------|
| **键** | 唯一标识符 |
| **标签** | 正向标签（例如「使用」） |
| **反向标签** | 反向标签（例如「被使用」） |
| **源类型** | 「起始」侧的卡片类型 |
| **目标类型** | 「目标」侧的卡片类型 |
| **基数** | n:m（多对多）或 1:n（一对多） |

点击 **+ 新建关系类型**创建关系，或点击现有关系编辑其标签和属性。

### 关系属性

某些关系带有额外的属性，您可以在每条具体的连接上设置这些属性，而不是在关系类型上设置。例如，内置的 **组织 → 应用** 关系（「使用」）带有 **使用类型** 属性——可在每条连接上将其设置为 **所有者**、**用户** 或 **利益相关者**。这样，您就可以通过单一关系类型来建模一个由某组织*拥有*、并被其他组织*使用*的应用。所选的值会在卡片的 **关系** 部分显示为彩色标签；可在添加关系时设置，或稍后通过关系行上的编辑图标设置。

给定的一对卡片类型之间只能存在一种关系类型，因此请使用这些属性来限定连接的含义，而不是为相同的源和目标创建第二种关系类型。

### 管理关系值

点击关系行上的**管理关系值**（标签）图标，即可编辑其「类型」属性的值。您可以：

- 向现有选择器**添加自己的值**——例如在所有者 / 用户 / 利益相关者之外新增一个使用类型。
- 通过**添加类型**，为尚无类型选择器的关系**添加全新的类型选择器**——包括内置关系。

内置值（所有者、用户、利益相关者、流向值等）已**锁定**：无法重命名、改色或删除。但您可以**隐藏**某个内置值，使其不再出现在卡片的选择器中——已设置的值仍会显示。您自己的值则完全可编辑和删除。

## EA原则

**EA原则**选项卡允许您定义管理组织IT环境的架构原则。这些原则作为战略护栏——例如「复用优先于购买优先于构建」或「如果购买，我们购买SaaS」。

每个原则有四个字段：

| 字段 | 描述 |
|------|------|
| **标题** | 原则的简洁名称 |
| **声明** | 原则阐述的内容 |
| **理由** | 为什么该原则很重要 |
| **影响** | 遵循该原则的实际后果 |

可以通过每张卡片上的切换开关单独**启用**或**停用**原则。

### 从原则目录导入

Turbo EA 内置了一个**精选的 10 条行业标准 EA 原则参考目录**,因此您无需从一张白纸开始。打开右上角的头像菜单,选择**参考目录 → 原则目录**。在那里您可以:

- 搜索并浏览内置原则(标题、描述、理由、影响)。
- 多选条目并点击**导入** — 所选原则将作为可完全编辑的标准条目出现在「EA 原则」选项卡中。
- 安全地重新导入:已经存在的原则(通过其稳定的目录 ID 识别)将被跳过,即使您已在本地重命名。这些条目在目录中会显示绿色「已导入」徽章。

将目录用作起点,然后根据您的组织调整每条原则的标题、陈述、理由和影响。

### 原则如何影响AI洞察

当您在[投资组合报告](../guide/reports.md#ai-portfolio-insights)中生成**AI投资组合洞察**时，所有活跃的原则都会被纳入分析。AI会根据每个原则评估您的投资组合数据并报告：

- 投资组合是否**符合**或**违反**该原则
- 具体的数据点作为证据
- 建议的纠正措施

例如，「购买SaaS」原则会导致AI标记本地部署或IaaS托管的应用程序，并建议云迁移优先级。

## 合规法规

**合规法规** 标签页管理 [GRC 合规扫描器](../guide/grc.md#compliance) 所依据的法规框架。默认启用六个框架：

| 法规 | 范围 |
|------|------|
| **欧盟人工智能法** | 投放欧盟市场的 AI/ML 系统所需满足的要求 |
| **GDPR** | 欧盟《通用数据保护条例》 |
| **NIS2** | 欧盟《网络与信息安全指令 2》 |
| **DORA** | 欧盟金融机构《数字运营韧性法案》 |
| **SOC 2** | AICPA 服务组织控制信任服务标准 |
| **ISO/IEC 27001** | 信息安全管理体系标准 |

对每一行，您可以：

- **启用 / 禁用** 该法规（通过开关）—— 禁用后，后续每次扫描都会跳过该框架，其发现也会从仪表板中排除。已有的发现会被保留（不会删除），以便您日后重新启用。
- **编辑** 标题、范围描述以及提供给 LLM 的提示上下文。
- 通过 **+ 新建法规** **添加自定义法规** —— 例如 HIPAA、内部政策或行业专属框架。自定义法规是一等公民：会在按法规的标签页中显示、参与总体合规分数，并支持发现上的所有相同操作（确认、接受、晋升为风险）。
- **删除** 自定义法规 —— 内建法规无法删除，只能禁用。

合规扫描器与晋升为风险的流程**即使在未配置 AI 提供商时也可用** —— 手动录入发现、状态过渡、晋升为风险的路径都保持可用。仅当您实际触发新的扫描时才需要 AI。

## 卡片布局编辑器

对于每种卡片类型，类型面板中的**布局**部分控制卡片详情页面的结构：

- **分区顺序** —— 拖动分区（描述、EOL、生命周期、层级、关系和自定义分区）重新排序
- **可见性** —— 隐藏与某种类型无关的分区
- **默认展开** —— 选择每个分区是否默认展开或折叠
- **列布局** —— 为每个自定义分区设置 1 列或 2 列
- **在分区之间移动字段** —— 使用字段的**移动**操作（位于其编辑和删除按钮旁）将其移至其他分区，并保留其配置

## 资源

**资源**选项卡管理每张卡片的**资源**选项卡中提供的两个列表：

- **链接类型** — 文档链接的类别（例如*文档*、*合同*、*安全*）。每个链接类型还带有一个显示在链接旁的**图标**。
- **文件类别** — 分配给上传文件附件的类别。

对于每个列表，您可以：

- **添加条目** — 提供一个键（存储在卡片上的小写标识符，创建后固定）、显示标签，以及（针对链接类型）一个图标。
- **编辑**任意条目的标签、图标、排序和各语言翻译，包括内置条目。
- 使用开关**启用 / 停用**条目 — 停用的条目会从选择器中消失，但卡片上的现有值会被保留。
- **删除**自定义条目 — 内置条目无法删除，只能停用。

内置的**合同**链接类型默认启用。两个列表都包含在**工作区迁移**中，因此您的自定义会在实例之间克隆。
