# Управление окончанием жизненного цикла (EOL)

Страница администрирования **EOL** (**Администрирование > Настройки > EOL**) помогает отслеживать жизненные циклы технологических продуктов, связывая ваши карточки с публичной базой данных [endoflife.date](https://endoflife.date/).

## Зачем отслеживать EOL?

Знание того, когда технологические продукты достигают окончания жизненного цикла или окончания поддержки, критически важно для:

- **Управление рисками** — неподдерживаемое программное обеспечение является угрозой безопасности
- **Планирование бюджета** — планирование миграций и обновлений до окончания поддержки
- **Соответствие требованиям** — многие нормативные акты требуют использования поддерживаемого программного обеспечения

## Массовый поиск

Функция массового поиска сканирует ваши карточки **Приложение** и **ИТ-компонент** и автоматически находит соответствующие продукты в базе данных endoflife.date.

### Запуск массового поиска

1. Перейдите в **Администрирование > Настройки > EOL**
2. Выберите тип карточки для сканирования (Приложение или ИТ-компонент)
3. Нажмите **Поиск**
4. Система выполняет **нечёткое сопоставление** с каталогом продуктов endoflife.date

### Просмотр результатов

Для каждой карточки поиск возвращает:

- **Оценка совпадения** (0–100%) — насколько точно имя карточки соответствует известному продукту
- **Название продукта** — найденный продукт endoflife.date
- **Доступные версии/циклы** — версии выпуска продукта с датами поддержки

### Фильтрация результатов

Используйте элементы управления фильтрами для фокусировки на:

- **Все элементы** — каждая просканированная карточка
- **Только несвязанные** — карточки, ещё не связанные с продуктом EOL
- **Уже связанные** — карточки, уже имеющие связь с EOL

Сводка статистики показывает: всего просканировано карточек, уже связано, не связано и найдено совпадений.

### Связывание карточек с продуктами

1. Просмотрите предложенное совпадение для каждой карточки
2. Выберите правильную **версию/цикл продукта** из выпадающего списка
3. Нажмите **Связать** для сохранения ассоциации

После связывания на странице деталей карточки отображается **раздел EOL** с:

- **Название и версия продукта**
- **Статус поддержки** — цветовая кодировка: Поддерживается (зелёный), Приближается EOL (оранжевый), Конец жизненного цикла (красный)
- **Ключевые даты** — дата выпуска, окончание активной поддержки, окончание поддержки безопасности, дата EOL

## Отчёт EOL

Связанные данные EOL используются в [Отчёте EOL](../guide/reports.ru.md), который предоставляет обзорную панель состояния поддержки вашего технологического ландшафта по всем связанным карточкам.
