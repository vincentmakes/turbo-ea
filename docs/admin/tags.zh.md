# 标签

**标签**功能（**管理 > 元模型 > 标签**标签页）允许您创建用户可以应用到卡片上的分类标签。标签按**标签组**组织，每个组拥有自己的选择模式、类型限制，以及一个可选的必填标志，该标志与审批工作流和数据质量评分集成。

## 标签组

标签组是一类标签的集合。例如，您可以创建「业务领域」、「合规框架」或「团队所有权」等组。

### 创建标签组

点击 **+ 新建标签组**并配置：

| 字段 | 描述 |
|------|------|
| **名称** | 在卡片详情、清单筛选和报告中显示的名称。 |
| **描述** | 可选的自由文本，仅管理员可见。 |
| **模式** | **单选** —— 每张卡片一个标签；**多选** —— 每张卡片多个标签。 |
| **必填** | 勾选后，此组会对其适用的每种卡片类型参与审批闸门和数据质量评分。详见下文的**必填标签组**章节。 |
| **限制类型** | 允许的卡片类型可选列表。为空表示该组适用于所有类型；否则，只有列出的类型才会在卡片详情、筛选和门户中看到该组。 |

### 管理标签

在每个组内，您可以添加单个标签：

1. 在标签组内点击 **+ 添加标签**。
2. 输入标签**名称**。
3. 可选设置**颜色**以便视觉区分 —— 颜色决定卡片详情、清单、报告和门户网站上的标签片背景色。

标签显示在卡片详情页的**标签**分区，具有相应权限的用户可以在此应用或移除它们。

## 类型限制

为标签组设置**限制类型**会在所有位置一次性收窄其作用范围：

- **卡片详情** —— 只有匹配的卡片类型才显示该组及其标签。
- **清单筛选侧边栏** —— 只有当清单视图按匹配类型筛选时，`TagPicker` 才会显示该组的标签片。
- **门户网站** —— 只有当门户展示匹配类型时，才向门户读者展示该组。
- **报告** —— 分组 / 筛选下拉菜单仅对匹配类型包含该组。

管理界面以小型标签片的形式在每个标签组上显示已指派的类型，便于一眼看清范围。

## 必填标签组

将标签组标记为**必填**后，它就成为治理要求：该组所适用的每张卡片必须携带该组中至少一个标签。

### 审批闸门

只要存在任何适用的必填标签组未被满足，卡片就无法进入**已审批**。尝试审批会返回 `approval_blocked_mandatory_missing` 错误，卡片详情页会列出缺失的组。两项细化保证了闸门的安全：

- 只有当标签组的**限制类型**列表为空或包含该卡片的类型时，该组才适用于此卡片。
- 一个**尚未配置任何标签**的必填组会被静默跳过 —— 这可防止因管理员配置不完整而出现无法通过的审批闸门。

补齐所需标签后，卡片即可正常审批。

### 数据质量贡献

适用的必填标签组同样会计入卡片的数据质量评分。每个已满足的组会与其他必填项（必填字段、必填关系端）一同提高评分，共同构成完整性计算。

### 视觉提示

必填组在管理列表和卡片详情的标签分区中都带有**必填**标签片。缺失的必填标签会出现在审批状态横幅和数据质量圆环的工具提示中，让用户准确知道需要补什么。

## 权限

| 权限 | 作用 |
|------|------|
| `tags.manage` | 在管理界面创建、编辑和删除标签组与标签，并在任何卡片上应用 / 移除标签，不受其他权限限制。 |
| `inventory.edit` + `card.edit` | 在用户拥有编辑权限的卡片上应用或移除标签（通过应用级角色或该卡片的相关方角色获得）。 |

`tags.manage` 默认授予 admin 角色。`inventory.edit` 归属于 admin、bpm_admin 和 member；`card.edit` 通过卡片自身的相关方角色分配授予。

查看者（viewer）可以**看到**标签，但不能修改它们。

## 标签出现的位置

- **卡片详情** —— 标签分区列出适用的组和当前附加的标签。必填组显示标签片；受限组仅在卡片类型匹配时出现。
- **清单筛选侧边栏** —— 分组的 `TagPicker` 允许按一个或多个标签筛选清单网格。组和标签按当前类型范围过滤。
- **报告** —— 组合、矩阵以及其他支持分组 / 筛选维度的报告都可按标签切片。
- **门户网站** —— 门户编辑者可以向匿名读者提供基于标签的筛选，让外部访客以相同方式切分公开架构全貌。
- **创建 / 编辑对话框** —— 创建新卡片时会出现同一个 `TagPicker`，便于从一开始就设置必填标签，这对必填组尤其有用。
