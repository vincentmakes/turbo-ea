# 架构规划

架构规划是 **EA 交付** 中的手动规划工具，用于对您的 IT 版图变更进行建模——为某个组织将一个应用替换为另一个、退役遗留系统或引入新平台——并以**一张变更前/后对比图**进行沟通。它提供与 TurboLens Architect 类似的成果，但完全不含 AI：每一项拟议变更都由您完全掌控。

成果是一张分层依赖视图（Layered Dependency View），在同一张图中展示当前状态与规划状态，并带有变更指示：

- **红色叉号** — 标记为移除的卡片或关系
- **绿色加号** — 新添加的卡片或关系
- **蓝色替换箭头** — 一次替换：继任卡片及其继承的连接

## 创建规划

打开 **EA 交付**，在某个举措上使用 **添加 → 新建架构规划**（也可以创建未关联的规划，稍后再关联）。规划分四步构建：

1. **业务目标**（可选）— 指定此变更支持的目标卡片。它们显示在图表的战略层，让每位干系人在看到「做什么」的同时看到「为什么」，并在提交规划时预填举措关联。
2. **范围与基线** — 选择一张或多张范围卡片（组织、业务能力、单个应用等）及依赖深度（1–3）。**捕获基线**会为周边版图拍摄快照，作为「变更前」视图。快照使图表保持稳定，即使实时库存发生变化；之后可用**刷新基线**重新捕获——目标已消失的规划变更会被标记。
3. **规划变更** — 使用工具箱中的变更操作：
    - **添加卡片** — 将现有卡片纳入视图，或提议一张全新的卡片（名称 + 类型）。
    - **移除卡片** — 将卡片标记为退役。其连接变为红色。
    - **替换卡片** — 选择要替换的卡片及其继任者（现有或新提议）。继任者继承前任的关系，以蓝色替换边显示；可用**移除关系**单独切断继承的关系。
    - **添加 / 移除关系** — 绘制新连接或切断现有连接。关系类型会根据元模型进行校验。
4. **实时预览** — 合并后的前/后对比图随规划实时更新。可随时保存规划；它会出现在举措的**交付物**区域。

## 了解影响

架构规划不仅是图表编辑器——规划时，**影响**面板会呈现架构层面的影响；同样的数据也显示在可共享的预览中，并写入已提交的 ADR：

- **差距分析** —— TOGAF 风格的新增／移除／变更／保留汇总。
- **影响／波及范围** —— 移除或替换卡片时会显示哪些内容依赖它（「*N 个应用、M 个接口依赖此卡片*」），来自该卡片自身的影响分析。
- **能力覆盖缺口** —— 若某业务能力在目标状态下失去其全部支持应用，则会被标记。
- **成本与风险差异** —— 预计年度成本的前→后（含差值），以及受影响卡片上未处理风险的数量。提议卡片会计入其预计成本，并在提交时写入所创建的卡片。

## 提交规划

草稿规划可以**提交**（需要「提交架构规划」权限）。提交将：

- 创建一张**举措**卡片（使用您选择的名称和起止日期），并关联所支持的目标，
- 创建选定的**提议卡片**和**关系**，并将每张新卡片关联到举措，
- 在被移除和被替换的卡片上盖上**生命周期终止**日期（举措结束日期），使生命周期报告和路线图反映规划，
- 可选地创建一份**架构决策记录草稿**，记录每项变更——包括被切断的关系（仅记录，绝不删除）。

!!! note
    提交绝不会归档或删除任何内容。被移除的卡片仅获得生命周期终止日期；实际退役仍是通过正常库存流程进行的、由人主导的步骤。

提交后，规划变为只读并链接到所创建的举措。

## 权限

| 权限 | 授予 |
|------|------|
| `arch_plans.view` | 查看架构规划 |
| `arch_plans.manage` | 创建、编辑和删除规划 |
| `arch_plans.commit` | 提交规划（创建举措、卡片、关系、ADR 草稿，盖生命周期终止日期） |

默认情况下，成员可以查看、管理和提交规划；查看者只能查看。
