# ArchLens Искусственный интеллект

Модуль **ArchLens** обеспечивает анализ вашего ландшафта корпоративной архитектуры на основе ИИ. Он использует настроенного поставщика ИИ для выполнения анализа поставщиков, обнаружения дубликатов, оценки модернизации и архитектурных рекомендаций.

!!! note
    ArchLens требует коммерческого поставщика ИИ (Anthropic Claude, OpenAI, DeepSeek или Google Gemini), настроенного в [Настройках ИИ](../admin/ai.md). Модуль автоматически доступен при настроенном ИИ.

!!! info "Credits"
    ArchLens основан на проекте с открытым исходным кодом [ArchLens](https://github.com/vinod-ea/archlens) от [Vinod](https://github.com/vinod-ea), выпущенном под лицензией MIT. Логика анализа была перенесена с Node.js на Python и нативно интегрирована в Turbo EA.

## Панель управления

Панель управления ArchLens предоставляет обзор анализа вашего ландшафта:

| Показатель | Описание |
|-----------|-------------|
| **Всего карточек** | Количество активных карточек в вашем портфеле |
| **Средн. качество** | Средний балл качества данных по всем карточкам |
| **Поставщики** | Количество проанализированных технологических поставщиков |
| **Кластеры дубликатов** | Количество выявленных групп дубликатов |
| **Модернизации** | Количество найденных возможностей модернизации |

Панель управления также показывает карточки, сгруппированные по типу, и выделяет основные проблемы качества.

## Анализ поставщиков

Анализ поставщиков использует ИИ для категоризации ваших технологических поставщиков по более чем 45 отраслевым категориям (например, CRM, ERP, облачная инфраструктура, безопасность).

**Как использовать:**

1. Перейдите в **ArchLens > Поставщики**
2. Нажмите **Запустить анализ**
3. ИИ обрабатывает ваш портфель поставщиков пакетами, категоризируя каждого поставщика с обоснованием
4. Результаты показывают разбивку по категориям и подробную таблицу поставщиков

Каждая запись поставщика включает категорию, подкатегорию, количество связанных приложений, общую годовую стоимость и обоснование ИИ для категоризации.

## Разрешение поставщиков

Разрешение поставщиков строит каноническую иерархию поставщиков, разрешая псевдонимы и определяя родительско-дочерние связи.

**Как использовать:**

1. Перейдите в **ArchLens > Разрешение**
2. Нажмите **Разрешить поставщиков**
3. ИИ определяет псевдонимы поставщиков (например, «MSFT» = «Microsoft»), материнские компании и группировки продуктов
4. Результаты показывают разрешённую иерархию с оценками уверенности

## Обнаружение дубликатов

Обнаружение дубликатов выявляет функциональные пересечения в вашем портфеле — карточки, которые выполняют одну и ту же или схожую бизнес-функцию.

**Как использовать:**

1. Перейдите в **ArchLens > Дубликаты**
2. Нажмите **Обнаружить дубликаты**
3. ИИ анализирует карточки Application, IT Component и Interface пакетами
4. Результаты показывают кластеры потенциальных дубликатов с доказательствами и рекомендациями

Для каждого кластера вы можете:

- **Подтвердить** — Отметить дубликат как подтверждённый для дальнейших действий
- **Расследовать** — Отметить для дальнейшего расследования
- **Отклонить** — Отклонить, если это не настоящий дубликат

## Оценка модернизации

Оценка модернизации оценивает карточки на предмет возможностей обновления на основе текущих технологических тенденций.

**Как использовать:**

1. Перейдите в **ArchLens > Дубликаты** (раздел «Модернизация»)
2. Выберите целевой тип карточки (Application, IT Component или Interface)
3. Нажмите **Оценить модернизацию**
4. Результаты показывают каждую карточку с типом модернизации, рекомендацией, уровнем усилий и приоритетом

## Architecture AI

Architecture AI — это трёхфазный диалоговый помощник, который генерирует архитектурные рекомендации на основе вашего существующего ландшафта.

**Как использовать:**

1. Перейдите в **ArchLens > Архитектор**
2. **Фаза 1** — Опишите ваше бизнес-требование (например, «Нам нужен портал самообслуживания для клиентов»). ИИ генерирует вопросы для уточнения бизнес-требований.
3. **Фаза 2** — Ответьте на вопросы Фазы 1. ИИ генерирует технические углублённые вопросы.
4. **Фаза 3** — Ответьте на вопросы Фазы 2. ИИ генерирует полную архитектурную рекомендацию, включающую:

| Раздел | Описание |
|--------|-------------|
| **Архитектурная диаграмма** | Интерактивная диаграмма Mermaid с масштабированием, загрузкой SVG и копированием кода |
| **Слои компонентов** | Организованы по архитектурному слою с классификацией существующий/новый/рекомендуемый |
| **Пробелы и рекомендации** | Пробелы в возможностях с рекомендациями рыночных продуктов, ранжированными по соответствию |
| **Интеграции** | Карта интеграций, показывающая потоки данных, протоколы и направления |
| **Риски и следующие шаги** | Оценка рисков с мерами по снижению и приоритизированными шагами внедрения |

## История анализов

Все запуски анализов отслеживаются в **ArchLens > История**, показывая:

- Тип анализа (анализ поставщиков, разрешение поставщиков, обнаружение дубликатов, модернизация, архитектор)
- Статус (выполняется, завершён, ошибка)
- Временные метки начала и завершения
- Сообщения об ошибках (при наличии)
