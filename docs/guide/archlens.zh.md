# ArchLens 人工智能分析

**ArchLens** 模块为您的企业架构全景提供人工智能驱动的分析。它使用您配置的 AI 提供商来执行供应商分析、重复检测、现代化评估和架构建议。

!!! note
    ArchLens 需要在 [AI 设置](../admin/ai.md) 中配置商业 AI 提供商（Anthropic Claude、OpenAI、DeepSeek 或 Google Gemini）。当 AI 配置完成后，该模块自动可用。

!!! info "Credits"
    ArchLens 基于 [Vinod](https://github.com/vinod-ea) 开发的开源项目 [ArchLens](https://github.com/vinod-ea/archlens)，以 MIT 许可证发布。分析逻辑已从 Node.js 移植到 Python，并原生集成到 Turbo EA 中。

## 仪表板

ArchLens 仪表板提供您的全景分析概览：

| 指标 | 描述 |
|------|------|
| **卡片总数** | 您投资组合中的活跃卡片数量 |
| **平均质量** | 所有卡片的平均数据质量评分 |
| **供应商** | 已分析的技术供应商数量 |
| **重复集群** | 已识别的重复分组数量 |
| **现代化** | 已发现的现代化机会数量 |

仪表板还按类型显示卡片分组，并突出显示主要质量问题。

## 供应商分析

供应商分析使用 AI 将您的技术供应商分类到 45 多个行业类别中（例如 CRM、ERP、云基础设施、安全）。

**使用方法：**

1. 导航到 **ArchLens > 供应商**
2. 点击 **运行分析**
3. AI 分批处理您的供应商组合，对每个供应商进行分类并提供推理
4. 结果显示类别细分和详细的供应商表格

每个供应商条目包括类别、子类别、关联应用数量、年度总成本以及 AI 对分类的推理。

## 供应商解析

供应商解析通过解析别名和识别父子关系来构建规范的供应商层次结构。

**使用方法：**

1. 导航到 **ArchLens > 解析**
2. 点击 **解析供应商**
3. AI 识别供应商别名（例如「MSFT」=「Microsoft」）、母公司和产品分组
4. 结果显示已解析的层次结构及置信度分数

## 重复检测

重复检测识别您投资组合中的功能重叠——服务于相同或类似业务目的的卡片。

**使用方法：**

1. 导航到 **ArchLens > 重复**
2. 点击 **检测重复**
3. AI 分批分析 Application、IT Component 和 Interface 卡片
4. 结果显示潜在重复的集群及证据和建议

对于每个集群，您可以：

- **确认** — 将重复标记为已确认以进行后续跟进
- **调查** — 标记为需要进一步调查
- **忽略** — 如果不是真正的重复则忽略

## 现代化评估

现代化评估根据当前技术趋势评估卡片的升级机会。

**使用方法：**

1. 导航到 **ArchLens > 重复**（现代化部分）
2. 选择目标卡片类型（Application、IT Component 或 Interface）
3. 点击 **评估现代化**
4. 结果显示每张卡片的现代化类型、建议、工作量级别和优先级

## Architecture AI

Architecture AI 是一个 3 阶段对话助手，根据您现有的全景生成架构建议。

**使用方法：**

1. 导航到 **ArchLens > 架构师**
2. **阶段 1** — 描述您的业务需求（例如「我们需要一个客户自助服务门户」）。AI 生成业务澄清问题。
3. **阶段 2** — 回答阶段 1 的问题。AI 生成技术深入问题。
4. **阶段 3** — 回答阶段 2 的问题。AI 生成完整的架构建议，包括：

| 部分 | 描述 |
|------|------|
| **架构图** | 交互式 Mermaid 图表，支持缩放、SVG 下载和代码复制 |
| **组件层** | 按架构层组织，标注现有/新建/推荐分类 |
| **差距与建议** | 能力差距及按匹配度排名的市场产品建议 |
| **集成** | 集成映射，显示数据流、协议和方向 |
| **风险与后续步骤** | 风险评估及缓解措施和优先实施步骤 |

## 分析历史

所有分析运行都在 **ArchLens > 历史** 中跟踪，显示：

- 分析类型（供应商分析、供应商解析、重复检测、现代化、架构师）
- 状态（运行中、已完成、失败）
- 开始和完成时间戳
- 错误消息（如有）
