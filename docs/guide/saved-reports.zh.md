# 已保存报告

Turbo EA 允许您**保存报告配置**，以便您可以快速返回特定视图，而无需每次重新配置筛选条件和轴。

## 保存报告

从任何报告页面（投资组合、能力地图、生命周期、依赖关系、成本、矩阵、数据质量或 EOL）：

1. 使用您期望的筛选条件、分组和轴选择配置报告
2. 点击报告工具栏中的**保存**按钮
3. 输入已保存报告的**名称**
4. 选择**可见性**：

| 可见性 | 谁可以看到 |
|--------|-----------|
| **私有** | 仅您自己 |
| **共享** | 您和您选择的特定用户 |
| **公开** | 平台的所有用户 |

对于共享报告，您可以向特定用户授予**编辑权限**，允许他们更新已保存的配置。

5. 点击**保存** —— 系统会自动从当前可视化捕获缩略图

## 已保存报告库

导航到**报告 > 已保存报告**浏览您有权访问的所有已保存报告。库以标签页组织显示缩略图预览：

- **我的报告** —— 您创建的报告
- **与我共享** —— 他人与您共享的报告
- **公开** —— 所有人可见的报告

### 操作

- **打开** —— 点击报告以加载其保存的配置
- **编辑** —— 更新名称、可见性或共享设置
- **复制** —— 使用新名称创建副本
- **删除** —— 移除已保存的报告（仅创建者或具有编辑权限的用户可以删除）

## 用你的 AI 助手构建自定义报告

除了内置的报告类型外，Turbo EA 还可以根据自然语言描述构建**完全自定义的报告**，方法是通过 **MCP 服务器**连接一个 AI 助手。

### 工作原理

1. 将 Turbo EA MCP 服务器连接到你的 AI 助手（例如 Claude Code）——请参阅 **MCP 集成**指南。
2. 用自然语言描述你想要的报告，例如「按业务关键性以饼图统计应用程序」或「按提供商分组的 IT 组件年度总成本」。
3. 助手会调用 `get_report_builder_schema` 读取你的实时元模型（卡片类型、字段、关系、标签），组装一份安全的报告**规范**，并通过 `preview_custom_report` 针对你的真实工作区数据进行预览——因此在保存任何内容之前你就能看到实际结果。
4. 当你满意时，助手会用 `create_saved_report` **发布**报告。它会出现在**已保存的报告**库中，并作为原生的交互式报告打开。

### 自定义报告能做什么

- **元模型感知**：你的卡片类型、子类型、字段、关系和标签会自动反映出来——无需编码。
- **分组与聚合**：按属性、子类型、生命周期阶段、标签组或关联卡片分组，并使用计数、求和、平均值、最小值或最大值进行度量。
- **筛选与遍历**：筛选源卡片，并可选地沿一个关系跳转到关联卡片。
- **多种可视化**：以表格、柱状图/条形图/饼图/环形图/散点图/树状图/折线图或 KPI 磁贴的形式呈现。
- **安全且受治理**：报告为只读，完全基于声明式规则运行（无代码、无 SQL），并且成本字段仍然受**查看成本**权限保护——与其他所有报告完全一致。

自定义报告的保存方式与其他报告相同，因此同样适用私有 / 共享 / 公开的可见性和共享选项。
