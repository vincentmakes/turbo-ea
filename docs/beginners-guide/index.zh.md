# 您在 Turbo EA 的前 30 天

您已经安装好了 Turbo EA。登录界面可以使用，演示数据能够加载，每个菜单项都展示了一些内容——而现在您正盯着一份空空如也的资产清单，琢磨着到底该从哪里开始。本指南就是为您准备的。

这是一份按步骤、有明确主张的演练，介绍大多数组织在 Turbo EA 上运行的第一个具体 EA 项目：把应用清单管理起来，并用它回答真实的组合问题。本指南刻意忽略了更高级的模块（风险登记册、合规、PPM、TurboLens AI）——这些只有在您的资产清单活起来之后才有用，而不是之前。

## 本指南的受众

- **企业架构师**——正在创建新的 EA 实践，或从电子表格、Confluence 或其他工具迁移过来。
- **解决方案架构师与应用负责人**——被要求「把数据填进 EA 工具」但缺乏上下文。
- **管理员**——为更大范围的推广做平台准备。

您需要 **admin** 角色（或至少具备 `admin.metamodel` 和 `inventory.edit` 权限）才能完成每一步。只读角色仍可受益——只是无法完成第 5 页中的元模型变更。

## 爬行 → 行走 → 奔跑 的演进路径

不要试图在第一周就建模整个企业。在 EA 工具上取得成功的团队都遵循分阶段的路径：

1. **爬行**——一个狭窄范围（一个业务领域、一个国家、一个平台）。一个卡片类型（应用）。每张卡片五个字段。在 50–200 张卡片上达到「够用」的数据水平。
2. **行走**——从内置目录中加入业务能力。将应用映射到能力。运行您的第一份组合分析。展示给一位干系人看。
3. **奔跑**——扩展到流程、接口、数据对象。增加更多自定义字段。打开更高级的模块。

本指南覆盖**爬行**阶段以及**行走**阶段的开始。读完之后，您将拥有一个可用的应用组合，附带 TIME 处置策略（**T**olerate 容忍 / **I**nvest 投资 / **M**igrate 迁移 / **E**liminate 淘汰），以及一份可以呈递给 CIO 的组合报告。

## 前置条件

- 一个运行中的 Turbo EA 实例（参见 [安装与设置](../getting-started/setup.md)）。
- 一个管理员账号（首位注册用户会自动成为管理员）。
- **首次使用者可选但推荐：** 使用 `SEED_DEMO=true` 启动一次，看看一份已填充的资产清单是什么样子（虚构公司 NexaTech Industries）。之后可以用 `RESET_DB=true` 重置，从真实数据重新开始。
- 对您想要首先建模的**业务领域**有一个大致的设想。「整个 IT」不算一个领域。

## 您暂时跳过的内容

这些都是强大的模块，但它们假设您已经有了一份填充好的资产清单。请先不要打开：

- **风险登记册**和**合规扫描**——在您有了应用和能力可以挂接风险之后才有用。
- **PPM**（项目组合管理）——在您有了值得追踪的项目流水线之后才有用。
- **TurboLens AI**（厂商分析、重复检测、架构师向导）——在您有了足够多的卡片供 AI 发现模式之后才有用。
