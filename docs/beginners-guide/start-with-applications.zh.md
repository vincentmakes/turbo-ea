# 从应用清单开始

Turbo EA 开箱即带 13 种卡片类型。您会忍不住想把它们全部填上。不要这么做。

**从应用开始**。在任何首次推广中，应用都是杠杆率最高的卡片类型：

- 它们最容易找到来源——IT 部门几乎总能在某个地方找到一份清单（CMDB、许可证跟踪、财务系统，甚至是一张电子表格）。
- 它们锚定了所有其他层——一旦有了应用，再映射到能力、流程和 IT 组件，就只是渐进的丰富工作，而不是从零开始的建模。
- 它们以最少的依赖驱动出第一份有用的报告（组合合理化）。

其他卡片类型可以稍后再说。常见的第二批是业务能力（第 4 页），然后是接口或数据对象。

## 「最小可用」是什么样子

对您初始范围内的每张应用卡片，只填以下字段，**仅此而已**：

| 字段 | 为何重要 | 数据来源 |
|-------|---------------|---------------------|
| **名称** | 身份标识。使用人们实际使用的名字，而不是许可证标签。 | 您现有的数据源 |
| **描述** | 一句话：这个应用为业务做什么？ | 与负责人访谈，或 AI 建议（参见[资产清单](../guide/inventory.md#ai-description-suggestions)） |
| **生命周期阶段** | 计划 / 引入 / 在用 / 退役中 / 生命周期结束 | CMDB，或与负责人访谈 |
| **业务负责人**（干系人） | 对该应用负责的人 | 组织架构图 |
| **成本——年度总额** | 用于组合报告和 TIME 公式 | 财务，或粗略估计 |

五个字段。仅此而已。数据质量环会显示约 50%，没关系——您可以在第二轮中细化。

!!! warning "请勿这么做"
    不要在第一轮就尝试填写**生命周期结束日期**、**厂商**、**技术栈**和 12 个自定义字段。您会在第 30 张卡片处崩溃。

## 填充资产清单的三种方式

挑选与您的数据源匹配的路径。您可以混合使用——批量导入，再手动修整长尾。

### 路径 A——Excel / CSV 导入（适合大多数起步）

如果您的应用清单在电子表格里（或可以从 CMDB 导出），这是最快的路径。**不要从零开始手工拼出表格**——让 Turbo EA 为您生成模板。

1. **先手动创建一张「示例」应用卡片**。进入**资产清单 → + 创建**，类型 = `Application`，名称取为类似 *「_TEMPLATE——稍后删除」*。把五个最少字段都填上（描述、生命周期、负责人、成本），这样导出文件里就有真实示例值。
2. **将资产清单按类型 = `Application` 过滤**，然后点击工具栏中的**导出**。您会得到一个 `.xlsx` 文件，里面有一行真实数据、每个字段对应一列——这就是您的模板。列标题与导入器期望的字段键一致。
3. **离线编辑该表格**：保留列结构，将这一行替换为您所有的真实应用，并删除末尾的示例行（或保留——导入完成后再从 Turbo EA 中删除这张卡片即可）。
4. **导入编辑好的文件**：**资产清单 → 导入**，拖入 `.xlsx`。校验报告会清楚地显示哪些行会创建新卡片、哪些会更新现有卡片（按名称或 ID 匹配）、哪些会失败。
5. 执行导入，然后归档 `_TEMPLATE` 卡片。

完整参考：[资产清单 → Excel 导入](../guide/inventory.md#excel-import)。

**首次导入小贴士：** 只包含上述五个最少字段，再加一列业务负责人邮箱（导入器会尝试匹配到已有用户）。其他都跳过。之后可以通过重复「导出—编辑—导入」循环，再做一次包含更多列的导入。

### 路径 B——ServiceNow 同步

如果您有 ServiceNow CMDB 并具有 API 管理权限，则集成可以直接拉取应用记录。

1. 进入**管理员 → ServiceNow 集成**。
2. 创建连接（URL、凭据——凭据加密存储）。
3. 定义映射：ServiceNow `cmdb_ci_business_app` → Turbo EA `Application`，并设定字段级规则。
4. 运行一次**拉取**同步。默认情况下，记录会落入**暂存区**，由管理员审核后再生效。

完整配置请参见[管理员 → ServiceNow 集成](../admin/servicenow.md)。把首次同步当作探索性运行——审阅拉取到了什么、改进映射，然后再真正运行。

### 路径 C——手动录入

适用于小型应用群（不超过 30 个应用）或不存在可用数据源的情况：

1. **资产清单** → **+ 创建**（右上角）。
2. 类型 = **Application**，填写名称和（可选的）描述。
3. 如果想要从网络搜索拉取一份起步描述，点击**用 AI 建议**。
4. 保存并继续。其余内容可以在卡片详情页中补充。

手动录入速度较慢，但产生的数据质量最高，因为每张卡片在录入时都经过了负责人确认。

## 把审批流作为质量关口

每张卡片都带有**审批状态**：草稿 → 已批准 →（在批准后被实质性编辑则变为损坏）。

实用的工作流：

1. 新卡片以**草稿**状态落地。架构师（您）做一次快速通读——名称正确、描述合理、生命周期无误。
2. 一旦最少字段填齐，**批准**该卡片。这向下游消费者表明该卡片可信。
3. 如果之后有人编辑了实质性字段，Turbo EA 会自动把状态翻转为**损坏**，直到重新批准。

按 `审批状态 = 已批准` 过滤资产清单，可以为本指南末尾的组合报告获得一份干净的视图。

!!! tip "最佳实践"
    在每天结束时分批批准。这会迫使您重新阅读您刚导入的内容，尽早抓住最严重的数据质量问题。

## 何时停止填充并继续前进

满足以下条件即可结束本页内容：

- 范围内的每个应用都有一张卡片。
- 每张卡片都填齐了五个最少字段。
- 集合的平均数据质量 **≥ 40%**。
- 至少 50% 的卡片已批准。

不要等待完美。前往下一页——[充分利用参考目录](leverage-reference-catalogues.md)——在映射好能力之后再回来补充。
