# Начните с инвентаря приложений

Turbo EA поставляется с 13 типами карточек «из коробки». У Вас возникнет соблазн заполнить их все. Не делайте этого.

**Начните с Приложений**. Приложения — это тип карточки с наибольшим рычагом в любом первом развёртывании:

- Их проще всего получить — у ИТ-отделов почти всегда есть список где-то (CMDB, трекер лицензий, финансовая система, даже электронная таблица).
- Они являются якорем для каждого другого слоя — когда у Вас есть Приложения, сопоставление с Возможностями, Процессами и ИТ-компонентами становится инкрементным обогащением, а не работой с нуля.
- Они дают первый полезный отчёт (Рационализация портфеля) с наименьшим количеством зависимостей.

Другие типы карточек идут позже. Распространённая вторая волна — это бизнес-возможности (страница 4), а затем Интерфейсы или Объекты данных.

## Как выглядит «минимально жизнеспособное»

Для каждой карточки Приложения в Вашей начальной области заполните эти поля и **только** эти поля:

| Поле | Почему это важно | Откуда это берётся |
|-------|---------------|---------------------|
| **Имя** | Идентичность. Используйте имя, которое люди реально используют, а не лицензионную метку. | Ваш существующий источник |
| **Описание** | Одно предложение: что это приложение делает для бизнеса? | Интервью с владельцем или AI-предложение (см. [Инвентаризация](../guide/inventory.md#ai-description-suggestions)) |
| **Фаза жизненного цикла** | Plan / Phase In / Active / Phase Out / End of Life | CMDB или интервью с владельцем |
| **Бизнес-владелец** (стейкхолдер) | Лицо, ответственное за приложение | Орг. схема |
| **Затраты — Всего годовых** | Используется Portfolio Report и формулой TIME | Финансы или приблизительная оценка |

Пять полей. И всё. Кольцо качества данных покажет ~50%, и это нормально — Вы сможете уточнить во втором проходе.

!!! warning "Не делайте этого"
    Не пытайтесь заполнить **Дату конца жизни**, **Поставщика**, **Технологический стек** и 12 пользовательских полей в первом проходе. Вы выгорите примерно на 30-й карточке.

## Три способа заполнить инвентарь

Выберите путь, соответствующий Вашему источнику данных. Вы можете их смешивать — импортировать основную массу, затем вручную исправить длинный хвост.

### Путь A — импорт Excel / CSV (рекомендуется для большинства стартов)

Если Ваши приложения живут в электронной таблице (или Вы можете экспортировать их из CMDB), это самый быстрый путь. **Не начинайте с того, чтобы собирать таблицу вручную** — пусть Turbo EA сам предоставит шаблон.

1. **Сначала создайте одну тестовую карточку приложения вручную**. Перейдите в **Инвентаризация → + Создать**, Тип = `Application`, назовите её, например, *«_TEMPLATE — удалить»*. Заполните пять минимальных полей (описание, жизненный цикл, владелец, стоимость), чтобы экспорт содержал реальные значения-примеры.
2. **Отфильтруйте инвентаризацию по Типу = `Application`** и нажмите **Экспорт** на панели инструментов. Вы получите файл `.xlsx` с одной строкой реальных данных и колонкой на каждое поле — это и есть Ваш шаблон. Заголовки столбцов совпадают с ключами полей, которые ожидает импортёр.
3. **Отредактируйте таблицу офлайн**: сохраните структуру столбцов, замените единственную строку всеми Вашими реальными приложениями и удалите тестовую строку в конце (или оставьте — после импорта Вы удалите эту карточку из Turbo EA).
4. **Импортируйте отредактированный файл**: **Инвентаризация → Импорт**, перетащите `.xlsx`. Отчёт валидации точно покажет, какие строки создадут новые карточки, какие обновят существующие (сопоставленные по имени или ID) и какие не пройдут.
5. Запустите импорт, затем заархивируйте карточку `_TEMPLATE`.

Полный справочник: [Инвентаризация → Импорт Excel](../guide/inventory.md#excel-import).

**Совет для первого импорта:** включите только пять минимальных полей плюс столбец для email бизнес-владельца (импортёр попытается сопоставить его с существующими пользователями). Пропустите всё остальное. Позже Вы можете сделать второй импорт с большим числом столбцов, повторив цикл «экспорт — редактирование — импорт».

### Путь B — синхронизация с ServiceNow

Если у Вас есть ServiceNow CMDB и админ-доступ к её API, интеграция извлекает записи Приложений напрямую.

1. Перейдите в **Админ → ServiceNow Integration**.
2. Создайте соединение (URL, учётные данные — учётные данные хранятся зашифрованными).
3. Определите сопоставление: ServiceNow `cmdb_ci_business_app` → Turbo EA `Application`, с правилами на уровне полей.
4. Запустите **pull**-синхронизацию. По умолчанию записи попадают в **staging**-область для проверки администратором перед применением.

Полную настройку см. в [Админ → ServiceNow Integration](../admin/servicenow.md). Относитесь к первой синхронизации как к исследовательской — просмотрите, что пришло, уточните сопоставление, затем запустите по-настоящему.

### Путь C — ручной ввод

Для небольших ландшафтов (менее ~30 приложений) или когда нет пригодного источника:

1. **Инвентаризация** → **+ Создать** (правый верхний угол).
2. Тип = **Application**, заполните Имя и (опционально) Описание.
3. Нажмите **Suggest with AI**, если хотите получить стартовое описание из веб-поиска.
4. Сохраните и двигайтесь дальше. Остальное Вы заполните со страницы деталей карточки.

Ручной ввод медленный, но даёт самые качественные данные, потому что каждая карточка касается рук владельца при вводе.

## Используйте рабочий процесс согласования как контроль качества

Каждая карточка несёт **Статус согласования**: Draft → Approved → (Broken, если существенно отредактировано после согласования).

Практичный рабочий процесс:

1. Новые карточки попадают как **Draft**. Архитектор (Вы) делает быстрый проход — имя верное, описание разумное, жизненный цикл правильный.
2. Как только минимальные поля заполнены, **согласуйте** карточку. Это сигнализирует потребителям, что карточке можно доверять.
3. Если кто-то позже отредактирует существенное поле, Turbo EA автоматически переключит статус в **Broken** до повторного согласования.

Отфильтруйте инвентарь по `Approval Status = Approved`, чтобы получить чистое представление для отчёта по портфелю в конце этого руководства.

!!! tip "Лучшая практика"
    Согласовывайте пакетами в конце каждого дня. Это вынуждает Вас перечитывать то, что Вы импортировали, и рано ловить худшие проблемы качества данных.

## Когда остановиться с заполнением и двигаться дальше

Вы закончили с этой страницей, когда:

- У каждого приложения в Вашей области есть карточка.
- У каждой карточки заполнены пять минимальных полей.
- Среднее качество данных по набору — **≥ 40%**.
- Минимум 50% карточек согласованы.

Не ждите совершенства. Переходите к следующей странице — [Используйте справочные каталоги](leverage-reference-catalogues.md) — и возвращайтесь обогащать после того, как сопоставите возможности.
