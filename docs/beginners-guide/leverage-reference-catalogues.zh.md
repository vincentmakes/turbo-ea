# 充分利用参考目录

这个阶段的经典错误是：花三周时间打磨一套定制的业务能力模型，再花两周与高管对齐，然后发现这套模型与您所在行业的其他公司有 80% 是相同的。

**不要从零开始建模。** Turbo EA 内置了三个经过精挑细选的目录，为您提供经过实战检验的起点，可以在数天而非数月内完成调整：

- **业务能力目录**——按行业（银行、零售、制造、保险、公共部门等）划分的多层能力层级结构，外加跨行业的宏观能力。
- **流程目录**——按行业划分的参考业务流程，可直接导入为 `BusinessProcess` 卡片。
- **价值流目录**——端到端的价值流，用于框定能力图谱。

本页聚焦于业务能力目录，因为它驱动了最后一页中的能力热力图。其他两个工作方式相同。

## 为何从能力开始

**业务能力**是*业务所做之事*，用稳定且与技术无关的语言来表达——例如「订单管理」、「客户准入」、「理赔处理」。能力多年来几乎不变；应用却时刻在变。这正是为什么应用到能力的映射是整个元模型中最有用的一种关系：

- 它让您可以问**「有多少应用支撑客户准入？」**——并发现冗余。
- 它让您可以问**「哪些能力依赖于某个老旧应用？」**——并发现脆弱性。
- 它在组织重组、厂商更换、云迁移之后依然有效。

不需要 500 个能力才能产生价值。您只需要范围内的 **20–60 个能力，2 到 3 层深**。

## 导入一份起步能力图

1. 在主菜单中（用户指南下方）导航到 **能力目录**。
2. 使用顶部的筛选器：
    - **行业**——选您的行业（或如果都不合适就选「跨行业」）。
    - **层级**——先显示 L1 和 L2。之后随时可以深入下去。
3. 浏览树形结构。展开几个分支，感受一下层级深度。
4. 勾选您想导入的能力。**选择会级联**：勾选一个 L1 会勾选其所有后代；勾选一个 L2 也会勾选其 L1 父级，以保持层级结构连贯。
5. 点击**从所选项创建卡片**。

Turbo EA 会为每个被勾选的节点创建一张 `BusinessCapability` 卡片，保留父子层级，并为每张卡片打上稳定的 `catalogueId`，使得重新导入是**幂等的**——两次运行导入不会产生重复项。

完整参考：[能力目录](../guide/capability-catalogue.md)。

!!! tip "最佳实践"
    选一棵子树，而不是整个目录。例如做销售领域的应用组合合理化，导入 L1 能力「销售与客户管理」及其 L2 子项通常已经足够——也就是 10–15 个能力，而不是 300 个。

## 应该建多深

合适的深度取决于您要用它做什么：

| 深度 | 适用场景 | 典型卡片数量 |
|-------|------------|--------------------|
| **仅 L1** | 高管层概览，非常小的范围 | 8–12 |
| **L1 + L2** | 首次推广的最佳点——一屏可见，在报告中有用 | 30–60 |
| **L1 + L2 + L3** | 详细的基于能力的规划，大型企业 | 100–250 |
| **L4 及以下** | 特定深度专题，不适合起步基线 | 视情况而定 |

第一轮请选择 **L1 + L2**。之后可以通过同一目录导入更多层级——幂等的重新导入会把它们整齐地放到现有父级之下。

## 关于流程和价值流的一点说明

**流程目录**和**价值流目录**的工作方式相同：筛选、勾选、批量创建。如果您的首个用例是应用组合合理化，可以暂时跳过它们——能力映射就足以驱动最后一页的分析。

以下情况会用到它们：

- 您从「合理化应用」迈向「优化订单到现金的价值流」。
- 您开始基于得到的 `BusinessProcess` 卡片构建 BPMN 流程图（参见 [BPM](../guide/bpm.md)）。

## 如果我的行业不在目录中怎么办？

两个选项：

1. **选最接近的行业**并修剪。「跨行业」条目（财务、人力、IT、采购）几乎适用于每一家公司。
2. **组合使用**——先导入「跨行业」，再从特定行业目录中补充几项。

无论如何，**先导入，后定制**。重命名一个已导入的能力或添加一个子项比从零开始打字快得多。而且您保留了 `catalogueId`，未来目录更新可以干净地合并。

!!! warning "请勿这么做"
    不要为能力或流程创建自定义卡片类型，只是为了「让它们更像自家的」。内置类型自带正确的字段、正确的关系类型和正确的报告——自定义的等价物则不会有这些。

## 继续前请验证

满足以下条件即可结束本页内容：

- 您范围内的能力图谱已经存在于资产清单中（按 类型 = `Business Capability` 过滤）。
- 层级结构完整——打开几个 L2 能力，检查父级面包屑是否显示了正确的 L1。
- 能力总数在 20 到 60 之间。

您还没有把任何应用映射到能力——那是最后一页的内容。首先，让我们给应用添加一个自定义字段，让分析真正有用。
