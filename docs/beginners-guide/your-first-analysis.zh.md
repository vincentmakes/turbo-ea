# 您的首次分析：应用合理化

这就是回报所在。您已经有了一份应用清单、一份能力图谱，以及一个 TIME 处置字段。现在将它们关联起来，产出两份报告，足以向 CIO 证明整个 EA 项目的价值：

- 一份**组合报告**，展示每个应用，按成本设定大小，按 TIME 处置着色。
- 一份**能力热力图**，展示您在哪里存在冗余（一个能力对应多个应用）和脆弱性（一个能力仅由单一应用支撑）。

## 步骤 1——把应用映射到能力

整个元模型中最有价值的关系是 **Application → Business Capability**（`supports` / `supported by`）。您将为范围内的每个应用都设置它。

### 批量路径：资产清单编辑模式

1. 进入**资产清单**，按 类型 = `Application` 过滤。
2. 确保关系列 **Business Capability** 可见（列标签页 → 关系）。
3. 在工具栏中切换到**网格编辑**模式。
4. 点击每一行的能力单元格，选择一个或多个能力。
5. 保存。

对于 50–200 个应用，这只需要一个下午和一杯咖啡。

### 逐卡片路径

对于需要较多判断的映射（或与应用负责人有工作坊的场景），打开每张 Application 卡片，使用**关系**分区。您将获得完整的选择器，包含搜索、层级预览以及设置关系属性的能力。

### 每个应用对应多少个能力？

| 映射数量 | 含义 |
|--------------|---------------|
| **0** | 未映射——您的资产清单不完整。过滤这些并修正。 |
| **1** | 干净理想的情况——该应用恰好支撑一个能力。 |
| **2–3** | 没问题——许多应用横跨几个相关能力。 |
| **4+** | 可疑——您可能把「使用其数据」与「支撑」混为一谈。重新核对。 |

!!! tip "最佳实践"
    第一遍映射快速而粗糙。第二遍——与应用负责人一起评审——才是让数据值得信赖的关键。请为两遍都做规划。

## 步骤 2——选择如何填充 TIME 字段

您有两个选项。选一个（或两者并用，以计算为默认值并对异常做手动覆盖）：

### 选项 A——手动录入 TIME（推荐用于第一遍）

您在上一页添加了一个 `single_select` 的 `timeDisposition` 字段。直接使用它。与应用负责人开一场一小时的工作坊，通常可以分类 30–50 个应用：

- **Tolerate**——能用、成本低、并非战略差异化要素。维持现状。
- **Invest**——具有战略意义、增长领域，资助其改进。
- **Migrate**——在规划周期内替换或迁移到新平台。
- **Eliminate**——重复、生命周期结束、退役。

可以使用资产清单的**网格编辑**模式，并显示 `timeDisposition` 列，以快速完成这项工作。

### 选项 B——通过公式计算 TIME

如果您希望得到一个供负责人随后校验的起步建议，[计算字段](../admin/calculations.md)功能可以根据成本和生命周期数据推导出 TIME 的默认值。

在 `Application` 类型的 `timeDisposition` 字段上的示例公式：

```
IF(lifecycle_endOfLife <= TODAY() + 365, "eliminate",
   IF(costTotalAnnual > 500000, "invest",
      IF(costTotalAnnual < 50000, "tolerate", "migrate")))
```

它的作用：

- 在一年内将达到生命周期结束的应用 → **Eliminate**。
- 高投入的战略应用 → **Invest**。
- 低成本的工具型应用 → **Tolerate**。
- 其他一切 → **Migrate**（默认值，需要人工评审）。

每次卡片被保存时该公式都会自动运行，Turbo EA 会将该字段标记为只读并附带「calculated」徽章，使用户不会意外偏离该规则。

!!! warning "请勿这么做"
    计算出的 TIME 是一个**起步假设**，而不是裁定结果。要么在采信前与负责人逐项评审，要么在工作坊完成后关闭计算，依赖手动录入。

混合模式：在构建资产清单时保持计算开启，工作坊时关闭计算，把字段切回手动编辑用于最终决策。

## 步骤 3——运行组合报告

1. 进入**报告 → 组合**。
2. 配置坐标轴：
    - **卡片类型**：`Application`
    - **X 轴**：技术适配度的度量指标（如有），否则使用成本拆分、年限或生命周期。
    - **Y 轴**：业务价值的度量指标（否则用 `costTotalAnnual` 代替）。
    - **大小**：`costTotalAnnual`——支出越大，气泡越大。
    - **颜色**：`timeDisposition`——这能让报告真正具备决策导向。
3. 把配置保存为一个命名视图（「应用组合——销售领域」），以便后续回来查看。

观察要点：

- **巨大的红色气泡**（高成本的淘汰候选）——您最快可获得的节约。
- **巨大的琥珀色气泡**（高成本的迁移候选）——您影响最深远的转型决策。
- **位于矩阵右上但并非绿色的聚类**——未获得相应投资的战略应用。

参考资料：[报告](../guide/reports.md)。

## 步骤 4——运行能力热力图

1. 进入**报告 → 能力图**。
2. 热力图展示您的业务能力层级结构，单元格颜色强度与**支撑该能力的应用数量**成正比。

观察要点：

- **热单元格**（每个能力对应多个应用）——候选冗余。应用组合合理化最常见的业务场景就在这里。
- **预期会有应用但呈现冷色的单元格**——映射缺失，或确实未被充分支撑的能力。
- **活跃分支中间的白色单元格**——未映射的应用，或未建模的能力。

参考资料：[报告 → 能力图](../guide/reports.md)。

## 步骤 5——汇报并迭代

您现在已经拥有一份站得住脚的组合视图。把这两份报告呈给销售 CIO（或您范围的负责人），并：

- 对成本最高的前 10 个应用确认 TIME 判断。
- 在热力图中识别前 3 个热单元格作为候选合理化项目。
- 把后续事项作为评论或待办记录到应用本身上——Turbo EA 会按卡片追踪。

就这么多。您已经在 Turbo EA 上拥有了一份可运作的 EA 实践。

## 接下来做什么

一旦您的应用组合活跃并受到信赖，以下都会成为高价值的下一步动作。在拥有填充好的资产清单之前，它们都不太有用——这就是本指南刻意推迟它们的原因。

| 模块 | 何时打开 | 在哪里找到 |
|--------|----------------|------------------|
| **风险登记册** | 当您准备好针对应用和能力跟踪架构风险时（TOGAF 阶段 G）。 | [风险登记册](../guide/risks.md) |
| **GRC / 合规** | 当您需要把应用和能力对照法规（GDPR、NIS2、欧盟 AI 法案、DORA、SOC 2、ISO 27001）进行映射时。 | [GRC](../guide/grc.md) |
| **PPM** | 当合理化决策转化为带预算、进度和状态报告的项目时。 | [PPM](../guide/ppm.md) |
| **TurboLens AI** | 当卡片数量足以让 AI 找到厂商重复、现代化候选与架构推荐时。 | [TurboLens](../guide/turbolens.md) |
| **BPM** | 当您准备好对位于应用之上的流程进行建模时。 | [BPM](../guide/bpm.md) |
| **图表** | 当您需要与资产清单保持同步的自由形式架构图时。 | [图表](../guide/diagrams.md) |
| **EA 交付** | 当您开始产出 TOGAF 风格的架构工作说明书和架构决策记录时。 | [EA 交付](../guide/delivery.md) |

欢迎使用 Turbo EA。
