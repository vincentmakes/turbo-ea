# 轻量化地定制元模型

Turbo EA 的元模型完全是**管理员可配置的**——每个卡片类型、字段、子类型、关系和干系人角色都是数据，而非代码。您会忍不住想重新设计它。**别这么做。**

成功的团队**只有在默认字段无法回答他们的问题时**才会定制元模型。失败的团队会把第一个月花在把 `Application` 重命名为 `Solution`、添加 30 个自定义字段上，最终从未走到一份可用的报告。

## 添加字段前的两问测试

在添加哪怕一个自定义字段之前，先问自己：

1. **我会用这个字段来过滤、分组或报告吗？** 如果不是，它属于描述或标签——不是字段。
2. **同一种类型的每张卡片都需要这个相同的答案吗？** 如果不是，它是一个关系或附件，不是字段。

如果两个问题都不能回答「是」，就不要加这个字段。

## 实战示例：添加一个 TIME 处置字段

对于应用组合合理化，您需要给每个应用一个单一决策：**T**olerate 容忍 / **I**nvest 投资 / **M**igrate 迁移 / **E**liminate 淘汰（即由 Gartner 推广的 **TIME** 框架）。内置元模型并未提供 `timeDisposition` 字段，所以这是为数不多需要添加自定义字段的合适场景之一。

我们将在 `Application` 类型上添加一个 `single_select` 字段，包含四个色彩编码的选项，权重为 1，以便它对数据质量做出贡献。

### 步骤 1——打开类型编辑器

1. 进入**管理员 → 元模型**。
2. 点击 **Application** 类型卡片。
3. 类型抽屉会在右侧打开。切换到**字段**标签页。

### 步骤 2——添加字段

1. 选择您希望字段所在的分区（或新建一个名为「组合决策」的分区）。
2. 在该分区中点击 **+ 添加字段**。
3. 填写：
    - **键**：`timeDisposition`  *（小驼峰命名，无空格，将成为 JSON 中的属性键）*
    - **标签**：*组合处置（TIME）*
    - **类型**：`single_select`
    - **权重**：`1`  *（对数据质量分数有贡献）*
    - **必填**：保持**关闭**——一旦必填，会阻止所有现有卡片被批准。
4. 添加四个选项：

    | 键 | 标签 | 颜色 |
    |-----|-------|--------|
    | `tolerate` | Tolerate | 灰色 / 中性 |
    | `invest` | Invest | 绿色 |
    | `migrate` | Migrate | 琥珀色 |
    | `eliminate` | Eliminate | 红色 |

5. 为标签和每个选项**添加翻译**，覆盖所有您支持的语言——[管理员 → 元模型](../admin/metamodel.md)第 4 页讲解了翻译编辑器。跳过这一步会让非英语用户看到原样的「timeDisposition」。
6. 保存。

### 步骤 3——确认其工作正常

1. 打开任意 Application 卡片。新字段会出现在其分区中，处于空值状态。
2. 选一个值，保存。数据质量环应当上升几个百分点。
3. 回到**资产清单**，新字段现在可在**列**标签页中选择，也可以作为过滤条件——您已经可以按 TIME 过滤应用。

就这么简单。一个字段，十分钟，立即有用。

## 备选方案：改用标签组

如果该值是信息性而非可查询的，那么**标签组**（管理员 → 标签）比自定义字段更轻——无需更改元模型，无需迁移，更易演进。在以下情况使用标签组：

- 该值是描述性的（「面向客户」、「仅内部」、「2024 年收购」）。
- 您可能会频繁添加新选项。
- 您不需要在过滤下拉框中使用它，能边输入边搜索的标签芯片即可。

在以下情况使用自定义字段：

- 您需要把该值用在组合报告的轴上（X、Y、颜色）。
- 您希望把它纳入数据质量的加权计算。
- 它是一份不会频繁变化的受控词表。

TIME 处置属于自定义字段阵营，因为我们将在下一页把它用作组合报告的颜色轴。

## 应避免的反模式

以下是首次推广中最常见的元模型错误：

!!! warning "不要重命名内置卡片类型"
    把 `Application` 重命名为 `Solution` 看起来整洁，但会破坏能力热力图、组合报告和各类目录都依赖的概念映射。如果您的组织把它们称为「Solutions」，请设置**标签**的翻译——底层的 `key` 仍然保持为 `Application`。

!!! warning "不要在第一天就添加 30 个自定义字段"
    每个自定义字段都会增加数据收集的摩擦，并稀释数据质量分数。先加一个字段，用一个月，再加下一个。

!!! warning "不要在第一天就让新字段成为 `required`"
    `Required` 会阻止每张没有该值的现有卡片被批准。等您为 80% 以上的卡片填好值**之后**，再把字段设为必填。

!!! warning "不要用自定义卡片类型替代自定义字段"
    「移动应用」应当是 `Application` 的一个子类型，而不是一个新卡片类型。新类型不会免费获得能力映射、组合报告或目录导入。

## 其他可能用到的轻量扩展

以下是常见的第二轮扩展，但**只在真的需要时再添加**：

| 需求 | 添加位置 | 类型 |
|------|-------------|------|
| 业务价值评分 | Application | `single_select`（高/中/低）——驱动组合报告的 Y 轴 |
| 技术适配评分 | Application | `single_select`——驱动 X 轴 |
| 云就绪度 | Application | `single_select`（就绪 / 需要重构 / 留在本地） |
| 失稳风险类别 | Application、IT Component | `single_select`（单点故障等） |
| 成本拆分 | Application | `cost` 字段，例如 `costRunTotalAnnual`、`costChangeTotalAnnual` |

每一项都通过了用于组合分析的两问测试。每一项也都很适合改成计算公式而非手动录入——这正是下一页的内容。
