# Ваши первые 30 дней с Turbo EA

Итак, Вы установили Turbo EA. Экран входа работает, демо-данные загружаются, каждый пункт меню что-то показывает — и теперь Вы смотрите на пустую инвентаризацию и думаете, с чего же реально начать. Это руководство для Вас.

Это последовательное, опинионированное пошаговое руководство по **первой конкретной EA-инициативе**, которую большинство организаций запускают на Turbo EA: взять под контроль инвентарь приложений и использовать его для ответа на реальные вопросы по портфелю. В нём намеренно не рассматриваются более продвинутые модули (Реестр рисков, Compliance, PPM, TurboLens AI) — они становятся полезными после того, как Ваша инвентаризация оживёт, а не до того.

## Для кого это руководство

- **Корпоративные архитекторы**, начинающие новую EA-практику или мигрирующие с электронных таблиц, Confluence или другого инструмента.
- **Архитекторы решений и владельцы приложений**, которых попросили «заполнить EA-инструмент» без особого контекста.
- **Администраторы**, готовящие платформу к более широкому развёртыванию.

Вам потребуется роль **admin** (или как минимум `admin.metamodel` и `inventory.edit`), чтобы пройти все шаги. Роли только для чтения тоже получат пользу — они просто не смогут вносить изменения в метамодель на странице 5.

## Дуга crawl → walk → run

Не пытайтесь смоделировать всё предприятие за первую неделю. Команды, преуспевающие в EA-инструментах, идут поэтапно:

1. **Crawl (Ползать)** — одна узкая область (бизнес-домен, страна, платформа). Один тип карточки (Приложения). Пять полей на карточку. Достичь «достаточно хороших» данных по 50–200 карточкам.
2. **Walk (Идти)** — добавить бизнес-возможности из встроенного каталога. Сопоставить приложения с возможностями. Запустить первый анализ портфеля. Показать его стейкхолдеру.
3. **Run (Бежать)** — расширить на процессы, интерфейсы, объекты данных. Добавить больше пользовательских полей. Открыть более продвинутые модули.

Это руководство охватывает **crawl** и начало **walk**. К концу его у Вас будет работающий портфель приложений с TIME-диспозицией (**T**olerate / **I**nvest / **M**igrate / **E**liminate — Терпеть / Инвестировать / Мигрировать / Ликвидировать) и Portfolio Report, который можно показать CIO.

## Предварительные требования

- Запущенный экземпляр Turbo EA (см. [Установка и настройка](../getting-started/setup.md)).
- Учётная запись администратора (первый зарегистрировавшийся пользователь автоматически становится администратором).
- **Необязательно, но рекомендуется новичкам:** один раз запустите стек с `SEED_DEMO=true`, чтобы увидеть, как выглядит заполненная инвентаризация (вымышленная компания NexaTech Industries). Затем можно сбросить с `RESET_DB=true` и начать с чистого листа на реальных данных.
- Примерное представление о **бизнес-домене**, который Вы хотите моделировать первым. «Вся ИТ» — это не домен.

## Что Вы пропустите — пока

Это мощные модули, но они предполагают, что у Вас уже есть заполненная инвентаризация. Не открывайте их сейчас:

- **Реестр рисков** и **сканирование Compliance** — полезны, когда у Вас уже есть приложения и возможности, к которым можно привязать риски.
- **PPM** (управление портфелем проектов) — полезен, когда есть пайплайн проектов, который стоит отслеживать.
- **TurboLens AI** (анализ поставщиков, обнаружение дубликатов, мастер Architect) — полезен, когда у Вас достаточно карточек, чтобы ИИ нашёл в них закономерности.

Короткие указания «что дальше» по каждому из них Вы найдёте на [последней странице](your-first-analysis.md) этого руководства.
