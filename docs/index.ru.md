# Введение в Turbo EA

### Что такое Turbo EA?

**Turbo EA** — это современная самостоятельно размещаемая платформа для **управления корпоративной архитектурой**. Она позволяет организациям документировать, визуализировать и управлять всеми компонентами бизнес- и технологической архитектуры в одном месте.

### Для кого это руководство?

Данное руководство предназначено для **всех пользователей Turbo EA** — корпоративных архитекторов, ИТ-менеджеров, бизнес-аналитиков, разработчиков и администраторов. Независимо от того, оцениваете ли вы платформу, управляете ИТ-ландшафтом организации в повседневном режиме или настраиваете систему как администратор, — вы найдёте здесь всю необходимую информацию. Для начала работы не требуются углублённые технические знания.

### Ключевые преимущества

- **Полная прозрачность**: Просматривайте все приложения, процессы, бизнес-способности и технологии организации на единой платформе.
- **Обоснованные решения**: Визуальные отчёты (портфолио, карты бизнес-способностей, зависимости, жизненный цикл, затраты и многое другое), которые помогают оценить текущее состояние технологической инфраструктуры.
- **Управление жизненным циклом**: Отслеживайте статус каждого технологического компонента на протяжении пяти фаз — от планирования до вывода из эксплуатации.
- **Совместная работа**: Несколько пользователей могут работать одновременно, с настраиваемыми ролями, назначением ответственных, комментариями, задачами и уведомлениями.
- **Описания с помощью ИИ**: Генерируйте описания карточек одним нажатием. Turbo EA объединяет веб-поиск с локальной или коммерческой LLM для создания типизированных описаний — с оценкой достоверности и ссылками на источники. Работает полностью на вашей инфраструктуре для обеспечения конфиденциальности или подключается к коммерческим провайдерам (OpenAI, Google Gemini, Anthropic Claude и другим). Полностью контролируется администратором: выбирайте типы карточек для ИИ-подсказок, поставщика поиска и модель.
- **Визуальные диаграммы**: Создавайте архитектурные диаграммы с помощью встроенного редактора DrawIO, полностью синхронизированного с вашим реестром карточек.
- **Моделирование бизнес-процессов**: Редактор процессных схем BPMN 2.0 с привязкой элементов, рабочими процессами утверждения и оценкой зрелости.
- **Интеграция с ServiceNow**: Двусторонняя синхронизация с ServiceNow CMDB для связи вашего EA-ландшафта с данными ИТ-операций.
- **Многоязычность**: Доступно на английском, испанском, французском, немецком, итальянском, португальском, китайском и русском языках.

### Ключевые понятия

| Термин | Значение |
|--------|----------|
| **Карточка** | Базовый элемент платформы. Представляет любой архитектурный компонент: приложение, процесс, бизнес-способность и т.д. |
| **Тип карточки** | Категория, к которой относится карточка (Приложение, Бизнес-процесс, Организация и т.д.) |
| **Связь** | Соединение между двумя карточками, описывающее их взаимоотношение (например, «использует», «зависит от», «является частью») |
| **Метамодель** | Структура, определяющая, какие типы карточек существуют, какие поля они содержат и как связаны между собой. Полностью настраивается администратором |
| **Жизненный цикл** | Временные фазы компонента: Планирование, Внедрение, Активный, Вывод, Конец жизненного цикла |
| **BPM** | Управление бизнес-процессами — моделирование и анализ бизнес-процессов |
| **SoAW** | Statement of Architecture Work — формальный документ TOGAF для архитектурных инициатив |
