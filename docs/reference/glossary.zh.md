# 术语表

| 术语 | 定义 |
|------|------|
| **ADR（架构决策记录）** | 记录重要架构决策的正式文档，包括背景、决策依据、后果和考虑过的替代方案。ADR 支持签署工作流和修订链 |
| **审批状态** | 卡片的审核状态：草稿、已批准、已变更或已拒绝。已批准的卡片在编辑后变为已变更 |
| **书签/已保存视图** | 清单中保存的筛选、列和排序配置，可一键重新加载 |
| **BPM** | 业务流程管理 —— 对业务流程进行建模、分析和改进的学科 |
| **BPM 行排序** | BPM 流程导航器中流程类型行（核心、支持、管理）的显示顺序，可通过拖拽行进行配置 |
| **BPMN** | 业务流程模型与标记法 —— 用于建模业务流程的标准标记法（2.0 版本） |
| **业务能力** | 组织能做什么，不论如何实现 |
| **计算** | 管理员定义的公式，在卡片保存时自动计算字段值 |
| **卡片** | Turbo EA 中的基本信息单元，代表任何架构组件 |
| **卡片类型** | 卡片所属的类别（例如应用程序、业务流程、组织） |
| **置信度评分** | 0-100% 的评级，表示 AI 生成的描述的可靠程度 |
| **成本行** | PPM 计划中的预算或实际成本条目（资本支出/运营支出），用于跟踪财务支出 |
| **数据质量** | 基于已填字段及其配置权重的 0-100% 完整度评分 |
| **文件附件** | 通过资源选项卡直接上传到卡片的二进制文件（PDF、DOCX、XLSX、图片，最大 10 MB） |
| **图表** | 使用嵌入式 DrawIO 编辑器创建的可视化架构图表 |
| **DrawIO** | 用于可视化架构图表的嵌入式开源绘图工具 |
| **企业架构（EA）** | 组织和记录组织的业务与技术结构的学科 |
| **EOL（生命周期终止）** | 技术产品失去供应商支持的日期。通过与 endoflife.date 集成进行跟踪 |
| **甘特图** | 带有水平条的可视时间线，显示项目的进度、持续时间和进展 |
| **项目** | 涉及架构变更的项目或计划 |
| **生命周期** | 组件经历的五个阶段：规划、引入、活跃、淘汰、生命周期结束 |
| **LLM** | 大型语言模型 —— 生成文本的 AI 模型（例如 Ollama、OpenAI、Anthropic Claude、Google Gemini） |
| **MCP** | Model Context Protocol —— 一种开放标准，允许 AI 工具（Claude、Copilot、Cursor）连接外部数据源。Turbo EA 的内置 MCP 服务器提供基于用户权限的只读 EA 数据访问 |
| **元模型** | 定义平台结构的数据驱动模型：卡片类型、字段、关系和角色 |
| **里程碑** | 项目时间表中的重要事件或完成点，在甘特图中显示为菱形指示器 |
| **通知** | 由系统事件触发的应用内或电子邮件提醒（待办分配、卡片更新、评论添加等） |
| **OData 数据源** | 保存的库存视图（书签）上提供的 JSON 数据源，供 Power BI 或 Excel 等外部工具使用 |
| **Ollama** | 用于在本地硬件上运行 LLM 的开源工具 |
| **投资组合** | 作为一个组管理的应用程序或技术的集合 |
| **PPM** | 项目组合管理——管理项目和计划组合的学科，包括预算、风险、任务和状态报告 |
| **参考编号** | 为 ADR 自动生成的顺序标识符（例如 ADR-001、ADR-002），提供唯一的可读标签 |
| **关系** | 两张卡片之间的连接，描述它们的关联方式（例如「使用」、「依赖于」、「运行于」） |
| **资源选项卡** | 卡片详情页上的选项卡，将架构决策记录、文件附件和文档链接整合在一个位置 |
| **修订版（ADR）** | 已签署 ADR 的新版本，继承前一版本的内容和卡片关联，修订号递增 |
| **RAG 状态** | PPM 状态报告中用于进度、成本和范围的红-黄-绿健康指标 |
| **风险评分** | 自动计算的值（概率 x 影响），量化项目风险的严重程度 |
| **已保存报告** | 持久化的报告配置，包含筛选条件、轴和可视化设置，可重新加载 |
| **分区** | 卡片详情页面中包含相关字段的可分组区域，按卡片类型可配置 |
| **签署人** | 被指定审核和签署 ADR 或 SoAW 文档的用户。签署工作流跟踪待签署和已完成的签名 |
| **SoAW** | 架构工作说明书 —— TOGAF 正式文档，定义项目的范围和交付物 |
| **SSO** | 单点登录 —— 通过身份提供商（Microsoft、Google、Okta、OIDC）使用企业凭据登录 |
| **子类型** | 卡片类型中的二级分类（例如应用程序有子类型：业务应用、微服务、AI 代理、部署）。每个子类型作为子模板，可以控制字段的可见性 |
| **子类型模板** | 特定子类型中哪些字段可见或隐藏的配置。管理员通过在元模型管理中点击子类型标签来进行配置 |
| **干系人** | 在卡片上具有特定角色的人员（例如应用所有者、技术所有者） |
| **问卷** | 针对特定卡片类型的数据维护问卷，用于从干系人处收集信息 |
| **标签/标签组** | 按组织的分类标签，支持单选或多选模式，可选择限制类型，并提供可选的必填标志，该标志会阻止审批并参与数据质量评分 |
| **必填标签组** | 被标记为必填的标签组。适用的卡片必须至少附加该组中的一个标签才能被审批；其满足情况会贡献到卡片的数据质量评分 |
| **欧盟人工智能法案语义检测** | TurboLens 的合规检查阶段，它请求 LLM 标记嵌入了 AI / ML 能力的卡片（LLM、推荐引擎、计算机视觉、评分、聊天机器人……），即使这些卡片未被显式分类为 `AI Agent` / `AI Model`。这类发现会被标注为 **AI 检测** |
| **初始风险 vs 残余风险** | 在风险登记簿中对每项风险捕获的两种评估。`初始`为未经缓解的概率 × 影响；`残余`为缓解之后的概率 × 影响，只要存在缓解计划就可编辑。两者均通过 4×4 矩阵推导出等级 |
| **风险编号** | 在创建风险时分配的单调递增、人类可读的标识符（`R-000123`）。它在提升后的发现按钮（**打开风险 R-000123**）和负责人关联 Todo 的描述中始终可见 |
| **TOGAF** | 开放组架构框架 —— 广泛使用的 EA 方法论。Turbo EA 的 SoAW 功能与 TOGAF 对齐 |
| **状态报告** | 通过 RAG 指标跟踪项目进度、成本和范围健康状况的月度 PPM 报告 |
| **财年** | 用于预算编制和财务报告的 12 个月期间。可通过管理 > 设置进行配置——起始月份（一月到十二月）决定 PPM 预算行的分组方式 |
| **门户网站** | 选定卡片的公开只读视图，通过唯一 URL 无需身份验证即可访问 |
| **工作分解结构 (WBS)** | 将项目范围分层分解为工作包的层级结构，包含开始/结束日期和完成跟踪 |
| **工作包** | 甘特图时间线中的任务逻辑分组，具有自己的开始/结束日期和完成百分比 |
| **AI 建议** | 通过将网络搜索结果与大型语言模型（LLM）结合生成的自动卡片描述 |
| **AI 判定** | 用户对 LLM 关于某张卡片是否含 AI 能力（`hasAiFeatures = true / false`）的确认或否决。该判定会跨重新扫描持久保留，从而避免 LLM 漂移悄然改变《欧盟人工智能法案》的适用范围 |
| **GRC** | 治理、风险与合规——位于 `/grc` 的统一工作区，含三个标签页（治理、风险、合规），整合 EA 原则、ADR、风险登记册以及安全与合规扫描器 |
| **G 阶段** | TOGAF ADM 的「实施治理」阶段，是风险登记册的术语与生命周期来源 |
| **风险登记册** | 与 TOGAF G 阶段对齐的全景架构风险登记册，位于 `/grc?tab=risk`。与 PPM 中以单个项目为范围的风险不同 |
| **风险责任人** | 对某项风险负责的用户。分配后会自动在该用户的待办页生成一条系统待办，并触发 `risk_assigned` 通知 |
| **缓解任务** | 附加在风险上、具有明确负责人的工作项，记录具体的缓解活动。可以是一次性的，也可以是周期性的（每日 / 每周 / 每月 / 每年）。周期性任务在完成时按日历正确地向前推进 |
| **缓解任务周期** | 缓解任务的一个排期实例。状态依次为 `scheduled` → `open` → `done` / `skipped`。开启时记录受指派人快照，关闭时记录责任人快照，使得即便经过多年的责任人轮换，审计也能回答相关问题 |
| **提前期（缓解任务）** | `due_date` 之前多少天将已排期的周期晋升为 `open` 并落入受指派人的待办列表。按单位提供智能默认值（每日 / 每周 / 每月 / 每年分别为 1 / 2 / 7 / 14 天），最大不超过周期长度的一半 |
| **合规发现** | 合规登记册中针对某项法规 × 条款的一行记录。可由审核员手工撰写，也可由 TurboLens AI 扫描产生；两种来源共享同一套生命周期，并可提升为风险 |
| **宏能力** | 能力目录中位于 L1 之上的 Level-0 分组。以 `BusinessCapability` 卡片形式落入清单，`attributes.capabilityLevel = "Macro"`，`catalogueId` 以 `MC-` 为前缀。将层级深度限制放宽至 6 |
| **分层依赖视图（LDV）** | Turbo EA 用于依赖图的自有表示法：卡片按四个 EA 层划分为泳道、按卡片类型着色，提议的卡片渲染为带虚线边框和绿色「NEW」徽章的节点。被依赖关系报告、卡片详情中的依赖关系部分以及 TurboLens Architect 的目标架构所使用 |
| **TIME（Tolerate / Invest / Migrate / Eliminate）** | 一种针对应用的四象限组合分类框架，由 Gartner 推广。每个应用被标记为一种处置策略：Tolerate（维持现状）、Invest（投资改进）、Migrate（替换或迁移）或 Eliminate（退役）。在 Turbo EA 中通常作为应用卡片类型上的 `single_select` 字段添加，并用作组合报告的颜色轴 |
| **应用组合合理化** | Turbo EA 上最常见的首个 EA 倡议：盘点范围内的应用，按业务价值与技术适配度对其分类，并赋予 TIME 处置策略，从而驱动合并、替换或退役决策 |
| **Crawl-Walk-Run** | EA 入门指南中推荐的分阶段落地模式。Crawl = 缩小范围、仅应用、每张卡片五个字段。Walk = 增加能力映射和首个组合分析。Run = 扩展到流程、接口、数据以及高级模块 |
